# Документ 2. CJM + UX Flow Specification v0.2
## AI-помощник по ремонту квартиры

Версия: 0.2

---

# 1. Назначение документа
Описывает пользовательский путь, UX-логику, ключевые экраны, состояния, вопросы, точки отвала, платные зоны, аналитику и формат результата для B2C AI-продукта в сфере ремонта квартиры.

Продуктовая формула Stage 1:
**Фото комнаты → стиль через карточки → AI-preview → ограниченный бесплатный результат → платный room pack → workspace «Мои комнаты».**

## 2.2. Новая логика v0.2
- Stage 1 = обновить гостиную/спальню без капитального ремонта.
- Stage 1B = расчёт примерной стоимости через Renovation Cost Engine.
- Stage 1.5 / Stage 2 = повторить интерьер с фото.
- Бесплатно = preview, а не полный результат. Платно = полный room pack.
- Style onboarding = название стиля + визуальный акцент + фото.
- Пользовательская структура = объект/квартира → комната → подпроект → версия.
- Flow до первого результата короткий.

# 3. Рамка продукта
**Делает в Stage 1:** загрузить фото гостиной/спальни; выбрать уровень вмешательства; выбрать направление через style cards; получить AI-preview; увидеть ограниченный preview товаров/идей; понять бюджетный диапазон; купить room pack; сохранить в workspace; вернуться, создать комнату/версию.
**Не делает в Stage 1:** полноценный дизайн-проект, рабочую документацию, точную смету, 3D-план, инженерные расчёты, ремонт под ключ как сервис, repeat-reference как основной, кухню/ванную как основные, полную товарную выдачу бесплатно, точную стоимость без базы rates.
**Главный UX-принцип:** первая ценность быстро. Не превращать старт в длинную анкету. Порядок: минимум входных данных → AI-preview → ограниченный бесплатный результат → понятный paywall → полный room pack → workspace.

# 4. Сегменты
- **A. «Хочу обновить комнату»** — главный сценарий Stage 1. Ценность: AI-preview, 3–5 идей, preview товаров, диапазон бюджета, платный room pack.
- **B. «Хочу понять стоимость»** — Stage 1B. Ценность: диапазон, структура затрат, работы, материалы, confidence, список уточнений. Критично: стоимость считает Cost Engine, не LLM.
- **C. «Хочу сделать часть сам»** — блок внутри room pack (безопасный план, простые действия, инструменты, предупреждения, «можно самому / нужен специалист»).
- **D. «Повторить интерьер с фото»** — не в Stage 1 (сложнее технически, выше ожидание совпадения). Fake-door + waitlist. Stage 1.5 / Stage 2.
- **E. «Ремонт под ключ»** — Stage 3 / Contractor Handoff (предварительная диагностика, вопросы подрядчику, ориентир бюджета, ТЗ, лид).

# 5. Уровни вмешательства
1. **Быстро освежить без ремонта:** текстиль, ковёр, свет, декор, растения, зеркало, постеры, хранение, перестановка. НЕ входит: пол/стены/электрика/сантехника/демонтаж. «Можно изменить вид комнаты без стройки и мастеров». Stage 1: да.
2. **Недорого обновить:** покраска одной стены, обои на акцентной, замена части мебели/штор, добавление света, ковёр, декор, частичная сборка. Может требовать мастера (карниз, сложный свет, сверление, полки). «Небольшое обновление с максимальным визуальным эффектом». Stage 1: да.
3. **Лёгкий косметический ремонт:** стены, пол, потолок, свет, мебель, часть работ мастера, материалы. «Предварительно оценим идею и бюджет, но для точного расчёта нужны размеры и состояние». Stage 1: частично.
4. **Ремонт под ключ:** в Stage 1 не как полноценный результат — только ориентир, вопросы подрядчику, список данных для сметы, предупреждение о специалисте.

# 6. Style Cards Onboarding
**Почему меняем:** эмоциональные подписи («светло и спокойно») — не стили. В v0.2 карточка = профессионально корректное направление + понятный акцент.
**Формат карточки:** фото + название стиля/направления + акцент + короткое объяснение + действия.
```
Скандинавский стиль
Акцент: светлая база, дерево, простые формы.
Подходит, если нужен лёгкий интерьер без визуального шума.
[Нравится] [Не нравится] [Похоже, но дешевле]
```
**Направления:** Скандинавский, Джапанди, Современный минимализм, Современная классика, Лофт, Mid-century modern, Contemporary, Эко-стиль, Soft minimal, Rental-friendly.
**Действия пользователя:** нравится / не нравится / похоже но дешевле / нравится цвет / мебель / свет / материал / атмосфера / слишком дорого / пусто / тёмно / ярко.
**Система извлекает `style_profile`:**
```json
{ "selected_style_ids": [], "liked_style_cards": [], "disliked_style_cards": [],
  "preferred_accents": [], "avoided_accents": [], "palette": "warm_neutral",
  "contrast_level": "low", "material_preferences": ["wood","textile","matte_surfaces"],
  "decor_level": "medium", "practicality_level": "high", "budget_feeling": "middle", "constraints": [] }
```

# 7. CJM верхнего уровня
Этапы: Потребность → Первый контакт → Быстрый старт → Фото и короткий бриф → Style cards → AI-preview → Ограниченный бесплатный результат → Paywall → Полный room pack → Workspace → Возврат/новая комната/новая версия.

| Этап | Риск отвала | UX-решение |
|---|---|---|
| Потребность | Непонятно, что продукт делает | Пример: фото → результат → товары |
| Первый контакт | Кажется генератором картинок | Товары и бюджет рядом с визуализацией |
| Быстрый старт | Много вопросов | Один комбинированный экран |
| Фото + бриф | Лень заполнять | Минимум полей |
| Style cards | Не понимает стили | Название + акцент + фото |
| AI-preview | Нереалистично | Честные ограничения |
| Free preview | Мало ценности | Показать структуру полного room pack |
| Paywall | Не понимает, за что платить | Блокировка конкретных полезных разделов |
| Paid room pack | Разочарование | Качество, актуальность, предупреждения |
| Workspace | Пользователь потерял результат | «Мои комнаты» после результата |

# 8. Укороченный UX-flow Stage 1

## Экран 1. Landing
За 5 сек объяснить, что делает продукт. Заголовок: «Обновите комнату с помощью AI: визуальная идея, товары, бюджет и план по фото». CTA: «Обновить комнату», «Рассчитать стоимость». Вторичные: «Пример результата», «Как это работает».
Показать пример: фото до → AI-preview → заблокированный блок товаров («3 из 18») → бюджетный диапазон → план → CTA «Открыть полный room pack». Не показывать: длинную анкету, 20 сценариев, обещание точной сметы/повтора Pinterest, сложные термины без фото.
События: `landing_viewed`, `main_cta_room_refresh_clicked`, `main_cta_budget_clicked`, `example_result_viewed`.

## Экран 2. Комната + цель + уровень (один экран)
Заголовок: «Что хотите сделать с комнатой?». Комната: гостиная/спальня (+ disabled coming soon: детская/прихожая/кухня/ванная). Цель: обновить комнату (default) / понять стоимость. Уровень: 1 освежить / 2 недорого обновить / 3 лёгкий косметический — с пояснением что входит, нужен ли мастер, тип результата.
События: `room_type_selected`, `goal_selected`, `intervention_level_selected`, `unsupported_room_clicked`.

## Экран 3. Фото + короткий бриф (один экран)
Фото: минимум 1, желательно 2–3 («из угла комнаты, где видно стены, пол, окно, основную мебель»). Город (доступность/доставка/расчёт/коэффициенты). Бюджет: до 30к / 30–70к / 70–150к / 150–300к / >300к / не знаю. Что оставить (мультивыбор): пол/стены/диван-кровать/шкаф/шторы/свет/всё можно менять/не знаю. Ограничения: дети/животные/под сдачу/нельзя сверлить/нельзя красить/легко убирать/маленькая/мало света.
Не спрашивать до preview: точные размеры стен, высоту потолков, розетки, материалы стен, состояние основания.
События: `photo_upload_started`, `photo_uploaded`, `photo_quality_failed`, `brief_completed`, `city_selected`, `budget_selected`, `constraints_selected`.

## Экран 4. Style cards
«Выберите визуальное направление». Карточка: фото + название + акцент + объяснение. Достаточно 3–5 карточек (не заставлять проходить 20).
События: `style_cards_started`, `style_card_liked`, `style_card_disliked`, `style_card_budget_version_requested`, `style_profile_generated`.

## Экран 5. AI-preview + ограниченный результат
Система: проверяет фото, анализирует комнату, определяет объекты, формирует style_profile, генерирует visual preview, показывает ограниченный результат.
- **Блок 1. AI понял комнату** (компактно): нашли (диван/окно/пол/шкаф), оставляем, предлагаем изменить (шторы/свет/ковёр/декор), [Изменить].
- **Блок 2. Визуальный preview** (1 вариант) + предупреждение «Это визуальная концепция, а не рабочий проект. Перед покупкой проверьте размеры».
- **Блок 3. Идеи изменений** — бесплатно 3–5.
- **Блок 4. Ограниченный preview товаров** — 2–3 товара/категории. «Показано 3 из 18. В полном room pack: 18 товаров, 3 альтернативы, цены/ссылки, чек-лист замеров, PDF».
- **Блок 5. Примерный бюджет** — верхнеуровневый диапазон; если есть Cost Engine — показать confidence.
События: `ai_preview_viewed`, `detected_objects_block_viewed`, `detected_objects_edited`, `free_product_preview_viewed`, `free_budget_preview_viewed`, `paywall_cta_clicked`.

## Экран 6. Paywall / полный room pack
Заголовок: «Откройте полный план комнаты». Открывается после оплаты: полный список изменений/товаров, 3 альтернативы по ключевым, цены, ссылки, дата актуальности, подробный бюджет, чек-лист замеров, пошаговый план, PDF, сохранение в «Мои комнаты», обновление цен, новая версия. Блокировка: показывать структуру, закрыть детали («Свет — 3 варианта подобраны [Открыть]»).
События: `paywall_viewed`, `paywall_product_block_viewed`, `payment_started`, `payment_completed`, `payment_failed`, `room_pack_unlocked`.

## Экран 7. Workspace / Мои комнаты
Показывать после сохранения/оплаты/возврата/регистрации. Структура: Моя квартира → комнаты (проект, версия, статус, бюджет, товары, дата цен) + «Добавить комнату». Действия: открыть/создать комнату/версию, обновить цены, заменить недоступные товары, скачать PDF, вернуться к оплате, удалить.
События: `workspace_viewed`, `property_created`, `room_created`, `room_project_opened`, `version_created`, `prices_refresh_clicked`, `pdf_downloaded_from_workspace`.

# 9. Сценарий «Понять стоимость» (Stage 1B)
**Принцип:** стоимость нельзя считать через LLM без базы. Опора: город, тип комнаты, площадь, уровень вмешательства, виды работ, стоимость работ/материалов, коэффициенты сложности, уровень неопределённости.
**Flow:** выбор «Рассчитать стоимость» → система берёт известные данные (фото/город/room_type/intervention_level/budget/ограничения/scope) → спрашивает недостающее (площадь, высота потолка, состояние стен/пола, нужны ли работы мастера, уровень материалов эконом/средний/комфорт) → AI предлагает scope (покраска, подготовка стены, обои, текстиль, светильник, сборка мебели, карниз, полка), пользователь подтверждает → Cost Engine считает диапазон → результат.
**Формат результата:** общий диапазон + работы + материалы + товары + доставка + запас + confidence + что уточнить.
**Confidence:** low (нет размеров/города, плохое фото) / medium (есть фото, город, уровень, примерная площадь) / high (есть размеры, подтверждённый scope, город, материалы, актуальные rates).
События: `cost_scenario_started`, `cost_area_entered`, `cost_scope_suggested`, `cost_scope_confirmed`, `cost_engine_calculated`, `cost_result_viewed`, `cost_confidence_viewed`, `cost_pdf_cta_clicked`.

# 10. Сценарий «План своими руками»
Часть room pack, не отдельный сценарий. Можно советовать: выбор/заказ мебели, замена текстиля, покраска простой стены, сборка мебели, лёгкий декор, хранение. Нельзя как инструкцию: перенос розеток, сложная электрика, сантехника, газ, несущие стены, вентиляция, сложный демонтаж.
События: `diy_section_viewed`, `specialist_warning_viewed`, `tool_list_viewed`.

# 11. Сценарий «Повторить интерьер с фото»
Не в Stage 1. Fake-door: «Повторить интерьер с фото — Скоро. Оставьте email». События: `reference_scenario_fake_door_viewed`, `reference_scenario_waitlist_joined`.

# 12. Free vs Paid
**Бесплатно:** зашёл → фото → style cards → AI-preview → 3–5 идей → 2–3 preview товара → общий бюджет → блокировка полного room pack. Цель: доказать, что продукт понял комнату, показать пользу, не выдать всё.
**Платно:** полный список товаров, 3 альтернативы, цены, ссылки, дата актуальности, подробный бюджет, чек-лист, план, PDF, сохранение, обновление цен.
**Платёжный триггер:** после AI-preview и ограниченного preview товаров (не раньше — не увидел ценность; не позже — иначе платить слабее).

# 13. Данные продукта
User, Property, Room, Room project, Result version, Uploaded image, Style profile, Detected object, Product preview, Cost estimate, Estimate line, Payment. (Поля — см. исходный документ; соответствуют схеме БД в `tech-spec-ts-stack.md` §4.)

# 14. Analytics Events (по группам)
Acquisition, Start flow, Photo/brief, Style cards, AI-preview, Free preview, Paywall, Paid room pack, Workspace, Cost Engine, Reference fake-door — полный список событий см. по экранам выше и в исходном документе §14.

# 15. Точки отвала и решения
Лендинг (не понял) → пример результата. Длинный flow → объединить экраны. Style cards (не понимает стили) → фото+название+акцент. Paywall (за что платить) → структура заблокированного, «3 из 18», список выгод. Недоверие к бюджету → показать основу расчёта + confidence + что уточнить. Потеря после результата → workspace «Мои комнаты».

# 16. Acceptance Criteria UX Stage 1
1. Пользователь проходит укороченный flow (10 шагов: landing → комната/цель/уровень → фото/бриф → style cards → AI-preview → free result → paywall → оплата/сохранение → workspace → открыть комнату).
2. Бесплатный результат ограничен (без полного списка товаров/цен/ссылок/альтернатив/чек-листа/PDF).
3. Платный room pack имеет явную ценность (полный список, 3 альтернативы, цены/ссылки, бюджет, чек-лист, план, PDF, сохранение/обновление).
4. Style cards подписаны корректно (название + акцент + объяснение + фото; не только эмоции).
5. Workspace поддерживает несколько комнат/версий.
6. Стоимость считается не LLM (Cost Engine или явная маркировка «очень грубый preview»). Для Stage 1B обязательны city coefficients, work rates, material rates, scope, confidence.

# 17. MVP UX Scope
**Stage 1:** landing; укороченный старт; гостиная/спальня; фото; бриф; style cards; AI-preview; ограниченный бесплатный результат; paywall; paid room pack; workspace; PDF/export; базовые товары; предупреждения.
**Stage 1B:** сценарий стоимости; Cost Engine; базы работ/материалов; городские коэффициенты; confidence; структура затрат.
**Не в Stage 1:** repeat-reference как полноценный; кухня; ванная; точная смета; чертежи; 3D-редактор; сложные инженерные советы; подрядчики; маркетплейс внутри.

# 18. Главный вывод
CJM строится вокруг быстрой доставки первой ценности и последующей продажи полного room pack.
Правильный Stage 1: **Landing → комната/цель/уровень → фото/бриф → style cards → AI-preview/free preview → paywall → paid room pack → workspace.**
Ключевое: (1) не делаем repeat-reference в Stage 1; (2) убираем лишние экраны; (3) не отдаём полный товарный список бесплатно; (4) style cards с названием стиля и акцентом; (5) пользователь ведёт несколько комнат/версий; (6) стоимость считает Cost Engine, не LLM.
