# 1. Суть продукта

B2C AI-помощник по обновлению и ремонту комнаты. По фото комнаты пользователь получает: визуальную идею
обновления, подбор **реальных покупаемых товаров из фидов маркетплейсов**, примерный бюджет, план действий.

Позиционирование: не «AI-дизайнер интерьера», а практический помощник, снижающий неопределённость ремонта.

Рынок запуска: **Россия** (понятен рынок, готовая товарная инфраструктура — фиды, измеримый поисковый спрос).
Второй рынок (после подтверждения модели): **Великобритания**. Архитектура **locale-agnostic** с первого дня:
валюта, источники фидов, партнёрские сети (РФ: Гдеслон и аналоги; UK: Awin/CJ), городские коэффициенты —
через абстракции, без хардкода рубля и российских магазинов.

Профиль основателя: solo, технический стратег, не пишет код руками (вся разработка через Claude Code).
Ограничение на сложность: минимум самописной инфраструктуры, максимум managed-подходов.

# 2. Бизнес-модель: три ступени монетизации (складываются, не заменяют друг друга)

## 2.1. Ступень 1. Affiliate (бесплатный продукт)
Всё бесплатно для пользователя. Доход — партнёрские отчисления с покупок по реферальным ссылкам.
- фото комнаты (или концепция с нуля) → AI-визуализация → бесплатный подбор **до N элементов (N=3–5, тест)**
  — реальные товары из фидов, похожие на элементы визуализации/фото;
- все подобранные товары **показываются открыто**: фото, цена, реф-ссылка, 2–3 альтернативы на позицию;
- источник дохода: сеть **Гдеслон** (и аналоги), средняя комиссия ~3% от выкупленного заказа (AliExpress,
  «Симфония мебели» и др., сотни магазинов).
- **Ключевое отличие от v0.2: товары и ссылки НЕ за paywall.** Открытые товары = двигатель affiliate.

## 2.2. Ступень 2. Платный «Комната целиком» (room pack v0.3)
Freemium-граница = **количество элементов подбора**.
- Бесплатно: до N (3–5) элементов подбора.
- Платно: полная комната — все элементы (обычно 8–20), по каждой 3 альтернативы, + сервисный слой:
  Renovation Cost Engine (не LLM), структура бюджета, чек-лист замеров, пошаговый план, границы DIY,
  PDF/export, workspace (обновление цен, версии).
- Оплата: **разовый платёж** (не подписка на старте). Affiliate-ссылки в платном паке тоже реферальные.
- **Ценообразование:** цена = 1/4–1/3 от работы дизайнера (якорь: подбор+смета у дизайнера РФ ~10–25 тыс. ₽).
  - Розница: базовая гипотеза **2 990 ₽**; тест лестницей 1 990 / 2 990 / 4 990 ₽; якорение «у дизайнера от
    12 000 ₽». Возможен облегчённый SKU (подбор без Cost Engine/плана) ~1 490 ₽ — проверить каннибализацию.
  - Опт (дизайнеры интерьера): **~990 ₽/комната при пакете от 10**. Экономит дизайнеру 2–4 ч ручного подбора
    (час 1–2 тыс. ₽). Нужны: batch-режим, выгрузка в таблицу/PDF, без брендинга в клиентских материалах
    (white-label — опция позже).

## 2.3. Ступень 3. Застройщики (vision, Stage 4)
После известности продукта: покупатель квартиры видит свою квартиру с мебелью (визуал + товары + проект +
смета); застройщик оформляет **одну ипотеку «квартира + ремонт»** (напр. 10 млн → 13 млн ₽); состав интерьера
фиксируется в договоре; исполнение — застройщик/партнёр-оператор. Монетизация: комиссия с застройщика
~2–3% с мебели, ~10% с работ, усреднённо **~5% с чека ремонта ~3 млн ₽ ≈ 150 тыс. ₽/клиент**. Оффер:
«ремонт по ипотечной ставке» (прямой конкуренции у механики в РФ нет). Не проектируется в Stage 1–2.
Смежный retail-сценарий: тот же room pack ступени 2, применённый к новостройке.

# 3. Привлечение трафика и юнит-экономика

## 3.1. Канал: платный трафик на дешёвых информационных запросах
Ядро: кластер «сделать ремонт самому» (~100–150 тыс/мес) + смежные (~100 тыс). Гипотеза CPC ~10 ₽.
Плюс кластеры прямого соответствия («как обновить спальню», «интерьер гостиной недорого», «что купить в
гостиную») — меньше объём, выше конверсия. «Ремонт под ключ» — отдельная посадка (fake-door на подрядчиков).

## 3.2. Целевая юнит-экономика ступени 1 (ГЛАВНАЯ falsifiable-гипотеза)
- CPC ~10 ₽; целевой CAC покупателя ~450 ₽ → 1 покупка на ~45 визитов (CR визит→покупка ~2,2%);
- средний чек покупки ~30 000 ₽ (один предмет мебели); доход 3% ≈ 900 ₽; целевая маржа ~2× CAC.
- Проверяется в Stage 0/1 на 50–100 тыс. ₽ **до масштабирования**. Метрики: CPC факт, CR визит→продукт,
  CR→клик по товару, CR клик→атрибутированная покупка, средний чек, % выплаты.
- Риски гипотезы: потери атрибуции (покупки в приложениях МП, позже/с другого устройства, cookie window);
  «оформлен» vs «выкуплен» (отмены/возвраты мебели); фактическая ставка ≠ средним 3%.

## 3.3. SEO/контент (второй канал)
Контентные страницы под тот же кластер («до/после», подборки по стилям с реальными товарами). Не блокер
Stage 1, но закладывается в архитектуру (SSR/SSG, программатик «стиль × комната × бюджет»).

# 4. Продуктовая механика (что меняется относительно v0.2)

## 4.1. Freemium-граница
- Было (v0.2): бесплатно preview 2–3 товаров, полный список/цены/ссылки — платно.
- Стало (v0.3): бесплатно **полноценный подбор до N (3–5) с ценами и реф-ссылками**; платно — комната
  целиком (все элементы) + сервисный слой (Cost Engine, чек-листы, план, PDF, workspace).

## 4.2. Два входных сценария подбора
1. **По фото пользователя:** фото → AI-визуализация → подбор похожих товаров к элементам визуализации.
2. **По элементам без генерации:** до N элементов с фото → прямой поиск похожих в базе фидов. Проще
   технически, отдельная точка входа «найти похожую мебель по фото».

## 4.3. Product matching: принятая стратегия
«**Генерация → поиск максимально похожего в большой базе фидов**» (не inpainting в Stage 1).
- промпты генерации ограничены «каталого-представимыми» объектами (типовые формы/категории из фидов);
- честный копирайтинг: «визуальная концепция; товары подобраны максимально похожие»;
- **метрика честности**: доля позиций с similarity выше порога (embedding-based) — считать с первого дня,
  предиктор refund'ов и доверия; целевой порог — Stage 0 (человеческая оценка «похоже/не похоже»).
- основа: image embeddings товаров (`embedding_image`) + embeddings элементов сгенерированного изображения;
  поиск ближайших по векторному индексу (pgvector).

## 4.4. Товарные данные
Источник: фиды **Гдеслон** (AliExpress, мебельные магазины сети, ~200+). Задачи: пайплайн загрузки/нормализации
(категории, размеры, цвета, материалы, style_tags); embeddings по фото и текстам; ресинк (цены, наличие,
статус оффера); фильтрация мусора (без нормальных фото/размеров — понижать/исключать).

## 4.5. Renovation Cost Engine
Архитектура без изменений (AI определяет scope → детерминированный движок считает диапазон по
work_rates/material_rates/city coefficients + confidence). Размещение: **часть платного продукта ступени 2**.
Бесплатный контур показывает только **товарный бюджет** (сумма цен подобранных товаров), без стоимости работ.
Правило «LLM не выдумывает цены» сохраняется (free — только фиды, paid — фиды + Cost Engine).

# 5. Roadmap v0.3
- **Stage 0 — Concierge/валидация:** вручную 30–100 комнат; проверить гипотезу юнит-экономики на малом
  платном трафике; эталонная выборка для метрики похожести; **kill-критерий**: CAC покупателя устойчиво
  > 2× дохода при отработанных креативах/семантике → модель ступени 1 не сходится, пересборка.
- **Stage 1 — Free Matching MVP (ступень 1):** фото → style cards → AI-preview → бесплатный подбор до N с
  реф-ссылками; вход «найти похожую мебель по фото»; лимиты/anti-abuse на генерации анонимов; аналитика
  полной воронки + клики/постбэки; workspace минимальный.
- **Stage 1B — Paid Room Pack (ступень 2, розница):** платный «комната целиком» + Cost Engine + чек-листы +
  план + PDF; приём платежей; базы Cost Engine по 5–10 городам.
- **Stage 1.5 — Designer Wholesale (B2B):** пакетные тарифы, batch, выгрузки; позже white-label, API.
- **Stage 2 — Materials Layer + расширение:** материалы (краска/обои/полы) с нормами расхода; кухня/ванная
  ограниченно; «повторить интерьер с фото» (reference matching).
- **Stage 3 — Contractor Handoff:** заявки мастерам, ТЗ, лиды бригадам.
- **Stage 4 — Developer Partnerships (ступень 3, vision):** конфигуратор «квартира + ремонт», операторы
  отделки, комиссия с чека. Только после подтверждения ступеней 1–2.
- **UK Expansion** (после РФ): Awin/CJ, локальные фиды (Wayfair, Dunelm, B&Q), фунт, локальные rates.

# 7. Риски v0.3
1. Гипотеза affiliate не подтверждается (атрибуция/чек/ставки) → Stage 0 kill-критерий + paid pack как
   обязательный второй поток.
2. Разрыв «визуализация ↔ товары» (ждут совпадения, получают «похожее») → каталого-представимая генерация,
   честный копирайтинг, similarity с 1-го дня.
3. Качество фидов (фото/размеры/наличие) → фильтрация, статусы, ресинк.
4. Стоимость бесплатного контура (генерация на каждого посетителя) → лимиты, кэш, дешёвые модели, генерация
   после захвата минимального контекста.
5. Интент-несоответствие (DIY хочет инструкции, продукт даёт товары) → контентные посадки, отдельные входы.
6. Ступень 3: исполнение зависит от контрагентов + репутация на платформе + регуляторика ипотеки → наценка
   как буфер, детализация только Stage 4.
7. LLM-цены: только из фидов/Cost Engine, никогда из головы модели.
8. Каннибализация (free настолько полезен, что paid не нужен) → приемлемо (affiliate), но мониторить.

# 10. Формула v0.3
- **Бесплатно:** фото → AI-идея → до N реальных товаров с ценами и ссылками → доход affiliate ~3%.
- **Платно:** комната целиком → полный подбор + расчёт стоимости + план + PDF → разовый платёж
  (розница ~2 990 ₽, дизайнеры ~990 ₽/комната оптом; якорь — 1/4–1/3 цены дизайнера).
- **Vision:** «квартира + ремонт в одной ипотеке» для застройщиков → ~5% с чека (~150 тыс. ₽/клиент).
- **Канал:** дешёвый инфо-трафик (DIY, ~10 ₽ CPC) + SEO.
- **Главная гипотеза:** 1 атрибутированная покупка на ~45 визитов при чеке ~30 000 ₽ → доход ≈ 2× CAC.
