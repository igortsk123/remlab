## Цель
Пользователь вставляет ссылку на товар → система тянет характеристики из интернета и заполняет
поля материала выбранного вида; всё редактируемо. При неудаче — чистая деградация к ручному вводу.

## Источник задачи
Роадмап `plans/calc-materials-roadmap.md` (К4), шаг 4 сценария (вставить ссылку → авто-характеристики).

## Скоуп — что входит
- **`app/api/calc/parse-link/route.ts`** — серверный роут (server-side fetch: без CORS, таймаут,
  user-agent, обработка ошибок/редиректов); принимает url → возвращает partial `MaterialSpec` + мету
  (название, цена, домен).
- **`lib/calc/link-parse.ts`** — извлечение: OG-мета + эвристики по виду (размеры/цена/упаковка из
  заголовка/характеристик страницы), нормализация единиц; уверенность полей.
- **UI:** `components/calc/MaterialParams.tsx` (доработка) — поле «вставить ссылку» → вызов parse →
  префилл полей (подсветить автозаполненные, дать поправить). Состояния: loading/ошибка/пусто →
  fallback на ручной ввод. Ссылка сохраняется в `productUrl` (→ позиция сметы с /go/ реф, К3).
- Тест: `tests/unit/calc-link-parse.test.ts` — извлечение на ОФФЛАЙН-фикстурах HTML (без сети в CI).

## Скоуп — что НЕ входит
- Полный парсер под все маркетплейсы (MVP + деградация); поиск «дешевле» (К6); визуализация (К5).
- Обход анти-бот-защиты «любой ценой» — если магазин режёт, честный fallback.
