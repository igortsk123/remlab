# MASTER-роадмап — «план из планов»: от фото к «влезет + вот так впишется»

> Верхнеуровневый план. Каждая фаза при подходе к ней разбивается на под-план (`plans/<slug>.md`),
> выполняется, помечается done. Этот файл — спина; детали живут в под-планах. Цели: `goals-one-photo-furnish-fit.md`.

## Северная звезда (из брифа v0.3)
B2C: пользователь грузит фото комнаты → получает AI-обновление **реальной мебелью из фидов, которая доказанно
ВЛЕЗЕТ**, в правильном масштабе, с affiliate-ссылками. **Ступень 1 — БЕСПЛАТНО** (доход ~3% affiliate).
Козырь vs First Chair: знаем И зону, И товар → **«влезет / велик на N см»** + вставка в масштабе.

## ЯДРО-ПЕТЛЯ (free-tier MVP — спина роадмапа)
`фото → [восприятие: объекты + зоны + свободно + масштаб] → пользователь: «освежить / заменить это»
→ [подбор: 3 реальных товара из фида, которые ВЛЕЗУТ] → [fit-check: вердикт «влезет/велик N»]
→ [вставка в масштабе: «вот так впишется»] → показать + реф-ссылки`
Всё остальное (высоты, окна, эркер, видео, платные тиры) — вокруг этой петли, не на её критическом пути.

## ПРИНЦИПЫ (почему так)
1. **Не доводить замер до идеала — собрать петлю.** ±10% принят владельцем + правка человеком → перфекционизм линейки не критичен. Замер улучшаем там, где он реально блокирует петлю.
2. **Vision = пиксели, мы = метрика.** Модель даёт ровные рёбра/маски; метр даём мы (A4+solvePnP). (Контур комнаты — так.)
3. **Лучший метод ПО ТИПУ объекта** (роутер kind→алго); кросс-чек/фьюжн только где спорно (окна). Не общий фьюжн на всё.
4. **Надёжно из 1 фото:** ширина+позиция footprint, площадь пола, масштаб. **Нужен видео:** истинная глубина, полная комната, за мебелью.
5. **Человек правит** имя/зону/замену — это часть продукта, не костыль.
6. **Человек-фолбэк (общий протокол, owner 2026-07-06):** на КАЖДЫЙ объект/зону — прогнать все методы + каталог +
   допуски → посчитать уверенность → неуверенно → отдать человеку **предзаполненное «вероятно X, Ш×Г×В — поправь»**
   → пересчитать (площадь/зону/fit) с правкой. Не гадать молча.
7. **Две площади пола (разные методы):** «свободный пол» (куда ставить, ничего не двигая) = из СЕГМЕНТАЦИИ видимого
   пола, объекты мерить не надо. «Полная площадь комнаты» (с полом под мебелью) = по КОНТУРУ СТЕН (замкнутая площадь),
   НЕ вычитанием/сложением объектов (накопление ошибки). Occluded участок стены-пола за мебелью → достроить прямой
   стеной / спросить человека / видео. Footprint объекта нужен точно только для ЗАМЕНЫ (там — человек-фолбэк).

## АРХИТЕКТУРА: геометрия = сменный слой «СЦЕНА» (owner-инсайт 2026-07-06)
Геометрию оформить как интерфейс **Scene**, дающий downstream: масштаб (A4), плоскость пола, `floor(u,v)`,
3D/глубину в пикселе, позу камеры, контур стен. Тогда:
- **PhotoScene** (сейчас): A4 + solvePnP → одна плоскость, видимое только.
- **VideoScene** (позже): A4 + мультивью (VGGT/MASt3R) → полное 3D, видно ЗА мебелью и вокруг.
**Весь пайплайн ВЫШЕ сцены (детекция, роутер, каталог, само-проверка, консенсус, человек-фолбэк, план, fit-check,
вставка) — ОДИН И ТОТ ЖЕ.** Видео-тир = замена Scene, НЕ новый пайплайн → точнее И покрывает то, что 1 фото не может
(истинная глубина footprint, полный контур, окна эркера). Слабые места фото = сильные у видео. Собирать через этот интерфейс.

## СТЕК 2026 (из ресёрча 7 агентов, 2026-07-06) — решения по узлам
- **0 Масштаб:** A4 + `solvePnP IPPE_SQUARE` (+refineVVS) — SOTA при эталоне, оставить. **+GeoCalib** (гравитация/вертикаль/фокус) для робастности стен. A4 = метрич.якорь в ОБОИХ тирах.
- **1 Глубина фото:** свитч GPU-глубины на **MoGe-2** (облако+фокус+нормали) или **Metric3D V2** (лучшие нормали); **Depth Anything 3** (на fal, нативно мультивью → мост к видео) — самый простой свитч. Масштаб фиксируем МНК по углам A4. DA-Small на CPU — фолбэк. **Нормали → плоскости стен напрямую.**
- **1' Видео:** **MapAnything (Apache-веса)** — нативно метрич., коммерч.чисто, ~$0.10/комн., 60-80 резких кадров. AMB3R — потолок точности (лицензия TBD). VGGT НЕ метрич.; MASt3R — non-commercial (нельзя). A4 держим как residual-якорь (1-3%).
- **2 Контур комнаты:** НЕ по колонкам. **RANSAC-прямая на видимую линию стена-пол каждой стены → углы = пересечение соседних стен (амодально, эркер сам) → гомография A4 → чистый метрич.полигон.** Сегментация стен: Mask2Former/OneFormer ADE20K. NonCuboidRoom (MIT) — опция; Layout Anything — SOTA, весов пока нет (мониторить).
- **3 Окна/стены/эркер:** плоскость стены **на линии плинтуса** ИЛИ **по нормалям** (2 сошедшихся пути) → углы = луч∩плоскость. Заменяет провалившийся depth-фит.
- **4 Детекция/маски/имена:** **LLMDet** (Apache, CPU, чистые ярлыки) вместо GD-tiny; **SAM 3** (fal $0.005, text/box/point) маски (или SAM 2.1 Apache self-host); **имя незнакомого — маленький VLM** (Qwen2.5-VL 3B/7B или Florence-2 MIT) → снимает бОльшую часть ручной правки имён. НЕ YOLO-World (AGPL/GPL).
- **5 Fit-check:** **детерминированная 2D-геометрия** — Shapely + Minkowski-буфер зазоров/проходов + NFP/поиск позиции; **OR-tools CP-SAT** для «что убрать чтобы влез». НЕ ML. Габариты нового товара = из ФИДА (точно); глубина закрытого объекта = прайор каталога + одно касание человека.
- **6 Вставка:** композитинг, НЕ генерация мебели. LaMa стереть (по SAM-маске) → вставить РЕАЛЬНОЕ фото товара в вычисленном масштабе (гомография пола) → перелайт (IC-Light/ObjectDrop/ZeroComp) → шов Flux Fill (low-denoise, только контур). **Движок=истина размеров, картинка=убеждалка** (генератор НЕ авторит размеры).
- **7 Подбор из фида:** **DINOv3 (визуал.сходство) + SigLIP 2 (стиль/текст)** поверх **pgvector+HNSW** (свой). Эмбеддить КРОП объекта. Фильтр по реальным размерам в том же SQL. Make-or-break = **нормализация картинок фида** (фон/кроп). generate→retrieve — индустр.стандарт.
- **Стратегия:** не биться с IKEA (LiDAR+свой каталог). Занять пустоту: **measured-fit из ОБЫЧНОГО фото + мульти-ритейлер affiliate + RU**. Моат: технич.(fit-из-фото) > структурный(нейтральность к ритейлеру) > данные(RU-фид).

## ПОРЯДОК (от ЦЕЛЕЙ + ресёрча) — ядро продукта строим на «достаточном» замере
Ключ: fit-check + подбор + вставка СТРОЯТСЯ СЕЙЧАС; апгрейды замера улучшают вход, но НЕ блокируют петлю.
1. **Ф1a Контур+план пола начисто** — RANSAC-стены + гомография A4 (дёшево, чинит «не совпадает», уже понятно как).
2. **Ф2 fit-check движок** (Shapely, детерминизм) — недостающее ЯДРО продукта (вердикт «влезет/велик N/что убрать»).
3. **Ф3 подбор из фида** (DINOv3+SigLIP2+pgvector) + **Ф4 вставка в масштабе** (композитинг) — замкнуть петлю.
4. **Ф5 правка человеком** (VLM пред-именует, флаги ведут) — обернуть петлю.
5. **Апгрейды замера** (MoGe-2/нормали, стены-плоскости, LLMDet/SAM3/VLM-имена) — итеративно, где реально помогает.
6. **Ф7 видео-тир** (MapAnything) — под механику C / платные.
Высоты/окна/эркер отдельными числами (`run_f3`) — заморозить как вторичное до тира 2-3 (кроме fit-нужного).

## Критерии успеха MVP (Ступень 1)
- [ ] Из фото: план пола (зоны+свободно) + масштаб, узнаваемо (фото+план).
- [ ] Вердикт «диван 1.5м велик для зоны 1.2м / влезет» — работает.
- [ ] 3 реальных товара из фида, что влезут, + реф-ссылки.
- [ ] Вставка выбранного в масштабе («вот так впишется»).
- [ ] Неуверенное — под правку человеком.
