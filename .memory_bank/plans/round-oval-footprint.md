## Цель
Ловить КРУГЛЫЕ/ОВАЛЬНЫЕ объекты (круглый стол и т.п.) и мерить footprint как эллипс (диаметры), а не писать
прямоугольником. Сейчас круглый стол = «105×60» (ширина по фронту × глубина каталога) — неверно, должно быть ~Ø106.

## Источник задачи
Owner 2026-07-07: «стол круглый, а пишет 105×60, он ~106×106 — определять круглость и достраивать по кругу/овалу».

## Метод (из ресёрча 2026-07-07)
- **Проекция круга в перспективе = ЭЛЛИПС.** При известной позе камеры (у нас из A4/solvePnP) эллипс на снимке
  back-проецируется в круг/эллипс на горизонтальной плоскости → истинные диаметры.
- **Классификация форма:** `cv2.fitEllipse` на контуре маски → (1) эксцентриситет (ma/MA близко к 1 = круг),
  (2) отклонение контура от эллипса (площадь контура / площадь эллипса ≈ 1 = эллипс, а не прямоугольник).
  Сравнить с 4-угольником (`approxPolyDP`): 4 угла + прямые = прямоугольный; иначе круг/овал.
- **Замер (прагматично):** столешница — горизонтальная плоскость на высоте h. Взять точки границы маски (или точки
  fitEllipse), **пересечь лучи с плоскостью z=h** (h = высота из каталога, стол ~74; позже — оценить из геометрии),
  получить метрич. точки → `cv2.fitEllipse` в метрике → большая/малая ось = диаметры. Круг → оси ≈ равны.
- Источники: [Pose from single circle (arXiv 1804.04922)](https://arxiv.org/abs/1804.04922),
  [Plane rectification using a circle](https://www.researchgate.net/publication/224649283),
  OpenCV fitEllipse + отклонение контура.

## Ограничения / нюансы
- 2-fold неоднозначность одиночного круга снимается **горизонтальностью плоскости + ground constraint** (у нас есть).
- Высота h столешницы: v1 — из каталога (стол 74, ±10% при ошибке h ~±7%). v2 — восстановить h из эллипса (cone∩plane).
- Нужна МАСКА объекта (SAM через fal) — контур для fitEllipse; для круглого стола маска ≈ столешница-эллипс.
- Ноги/пьедестал в маске ниже столешницы → брать точки границы эллипса (рим), не всю маску.
