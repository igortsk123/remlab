## Цель
Свести ВСЕ наработки сессии в один пайплайн с **маршрутизацией по типу объекта** (на полу → алго A, на стене →
алго B, …) + **фьюжн методов** (голоса нескольких методов, согласие→уверенность). Ничего не выкидываем — каждый
метод как инструмент, применяется там, где силён. Итог: узнаваемый результат (фото+план) + честная уверенность.

## Источник задачи
Owner 2026-07-06: «объедини всё в единый большой пайплайн; зафиксируй какие методы для чего работали хорошо/плохо;
в зависимости от объекта (пол/стена дальняя/…) использовать разные алгоритмы; контур сейчас не совпадает с комнатой».

## ЕДИНЫЙ ПАЙПЛАЙН (маршрутизация по типу + фьюжн)
**Ст.0 — Фундамент:** A4 → solvePnP → масштаб + плоскость пола + FOV. (нет A4 → просить переснять).
**Ст.1 — Детекция+роутер:** Grounding DINO → рамки+ярлыки → `canon`→класс→**KIND**; перевод имени (авто-словарь);
крупный незнакомец с низкой conf → OpenAI-vision имя (фолбэк).
**Ст.2 — Замер ПО ТИПУ (маршрутизация):**
- **FLOOR (стул/стол/диван/радиатор/прибор на полу):** ширина+позиция = проекция низа на пол; высота = ГЕОМЕТРИЯ
  (⊕ depth-голос); footprint-глубина = каталог (низкая conf) / видео-тир.
- **SURFACE (монитор/чашка на столе):** containment → мерить от поверхности; мелочь — детект, но НЕ рисуем (счётчик).
- **WALL (окно/дверь/зеркало/картина):** плоскость стены ⊕ фронталь → согласие=conf, расхождение=флаг; потолок-чек.
  Угловой эркер = отдельная **пер-стеновая геометрия** (каждое окно на своей грани — TODO).
- **STRUCTURE (стены/пол/потолок):** SegFormer → площадь пола, контур стена-пол, высота потолка.
**Ст.3 — Фьюжн+уверенность:** на каждый размер — голоса применимых методов; согласны→взвеш.среднее, высокая conf;
расходятся→ближе к каталогу/геометрии, низкая conf + флаг. Каталог = жёсткие границы + фолбэк.
**Ст.4 — Относит. сверка:** консенсус одинаковых (медиана); порядок высот (уже из общей floor-калибровки).
**Ст.5 — Вывод:** узнаваемое ФОТО + связанный цветом ПЛАН сверху + флаги «проверить» → человек правит имя/глубину/замену.
**ВИДЕО-ТИР (отдельно):** мультивью (VGGT/MASt3R + A4) → истинная глубина/полная комната/за мебелью.

## Надёжно из 1 фото vs нужен ВИДЕО
- **1 фото надёжно:** ширина+позиция+высота напольных; площадь пола; видимый размах стен.
- **1 фото слабо (→ каталог/оценка + флаг):** footprint-глубина; угловые окна эркера; зад объектов.
- **Видео:** истинная глубина, полная комната, за мебелью.

## ПРОВЕРКА ПОИСКОМ (SOTA-2026, 2026-07-06) — апгрейды
- **Single-view metrology подтверждён + лимит назван:** точен только когда эталон и объект в ОДНОЙ плоскости и на
  схожем расстоянии → A4-на-полу отлично для пола, хуже для окон/дальних стен (природа метода, не баг).
- **Глубина: Depth Anything V2 Small → UniDepth V2 / Metric3D V2** — метрика лучше + НОРМАЛИ поверхностей (нормали
  задают плоскости пола/стен → окна/стены без хрупкого depth-фита).
- **Контур комнаты: не сглаживать сегментацию, а vanishing points ИЛИ обученная разметка «Layout Anything»** (дек-2025,
  non-cuboid, держит оклюзию). Настоящий фикс «контур не совпадает».
- **Стены/окна эркера: vanishing points** (перенос масштаба на плоскость стены) — принципиально вместо depth-подгонки.
- **Видео: VGGT → AMB3R / MapAnything** (feed-forward, метрика сразу; A4 держим для робастности).
- **Детекция: Grounding DINO 1.6 / YOLO-World** (апгрейд опционален — локализация уже ок).
- **Стратегия (owner-флаг):** IKEA Place/Kreativ уже делают measured-fit (Kreativ: LiDAR → удалить мебель → примерить, 98%).
  «measured fit» сам по себе НЕ уникален → козырь переточить: **от ФОТО/ВИДЕО без AR/LiDAR/приложения + мульти-ритейлер
  affiliate + РФ**. (Зафиксировать и в `goals-...`/`market`.)

## Ф1a КОНТУР — попытка RANSAC (2026-07-07): частично, нужен wall-instance seg
Реализовал (в `run_plan.py`): точки линии стена-пол (метрич.) → **секвенциальный RANSAC на прямые стен** → пересечение
соседних → углы → гомография A4. ПЛЮС **наложение контура обратно на фото** (голубая линия) — отличный инструмент
СВЕРКИ (сразу видно, совпал контур или нет). РЕЗУЛЬТАТ честно: на перспективных фото с эркером RANSAC **ненадёжен** —
эркерные грани короткие/редкие, пересечения улетают (room2 давал 32см, room1 435см, линия резала комнату по диагонали).
→ **защита-фолбэк на оболочку** (авто-отбраковка вырожденного по габаритам И по площади: контур должен совпадать
с площадью пола ±; room1 249×313, room2 230×306). Наложение-на-фото — постоянная проверка контура.
**ИТОГ (2026-07-07): CPU-геометрия исчерпана** — пробовал per-column, RANSAC(угол-порядок), RANSAC(imgx+обрезка
пересечений): все мимо на эркере (точки линии стена-пол из Depth/сегментации шумные, короткие грани не набираются,
линии крест-накрест — наложение это доказало). ADE20K-«стена» = stuff, instance-seg грани НЕ делит. Чистый ВОГНУТЫЙ
контур требует **нормалей поверхностей** (кластеризация wall-пикселей по нормали → плоскости граней) из сильной
метрич-depth модели (**Metric3D V2 / MoGe-2, GPU**) — не на fal готово, нужен self-host/RunPod. Это отдельный infra-шаг,
делать вместе с апгрейдом depth. ПОКА: устойчивая оболочка (выпуклая, эркер-вогнутость не ловит) — приемлемо для fit-check.

## Слабые места к доработке (приоритет)
1. **Контур комнаты не совпадает с фото** (owner) — зубчатый, выпуклый теряет форму. Решение: упорядочить точки
   стена-пол по обходу (не по image-x), сгладить, замкнуть по рамке; или взять floor-wall рёбра как в старом
   gpt-5.5-подходе (`run_plan.py` архив). Проверять наложением на фото.
2. **Угловой эркер (окна):** пер-стеновая геометрия — каждое окно на своей грани стены (не одна плоскость на все).
3. **Footprint-глубина:** честно каталог+флаг до видео-тира.
