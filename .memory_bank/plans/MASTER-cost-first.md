# МАСТЕР-ПЛАН v0.4 — «Смета-first»

> Решение владельца 2026-07-11 (ADR-0016): «строим продукт полностью в этой концепции».
> Заменяет прежние мастер-планы (commercial-master-plan Э0–Э8 и MASTER-roadmap Ф0–Ф8) —
> их живые части поглощены этапами ниже, остальное в архиве (таблица в конце).

## Концепция (утверждённый пользовательский сценарий)

**Скелет:** посчитал → получил смету-список → сохранил → купил по своим же ссылкам
(нам комиссия) → хвосты: «как будет выглядеть» (визуализация по фото) и «найти мастера» (лид).

**Вход А — «уже делает ремонт»** (запросы-калькуляторы: «сколько нужно обоев» 22.5k,
«калькулятор плитки» 15k, суммарно ~70–90k/мес; клик ~22–30 ₽):
калькулятор БЕЗ анкет → мгновенный ответ («13 рулонов с запасом») → «какие обои берёте?»:
СВОЯ ссылка (превращаем в реферальную, цена для него не меняется) или наши варианты →
«Сохранить смету» → **чек-лист по постоянной ссылке** (живой документ: позиции, количества,
сумма; открывается всегда, редактируется, к нему возвращаются). Обогащение ИИ: по вставленной
ссылке агент подбирает сопутствующее С ТОГО ЖЕ сайта («клей не забудь — поможем, подберём»)
и, где релевантно, «подберём дешевле, если найдём» (сравнение по магазинам). Покупает по
списку → комиссия. Внизу ненавязчиво: визуализация, позже мастер.
⚠️ Правило входа А: сначала ЧИСЛО, всё продуктовое — после ответа (иначе уйдёт к обычному
калькулятору). Козырь: «не вводи размеры — сфоткай комнату, посчитаем сами».

**Вход Б — «только присматривается»** («сколько стоит ремонт квартиры» 52k, «бюджетный/
косметический/хрущёвка» ~30k+; клик ~25–36 ₽): три вопроса (площадь ИЛИ фото → глубина
ремонта: освежить/обновить/капитально → стиль-свайп) → вилка «Эконом ~X · Средний ~Y ·
Получше ~Z» с разбивкой материалы/работа («своими руками — минус работа») → выбирает →
ТА ЖЕ смета-список (наши реф-позиции) → дальше как вход А.

**Почему один продукт:** один человек на разных стадиях (сегодня присматривается — через
месяц клеит). Ядро одно — «Смета-лист»; различаются только первые экраны. Реклама уже
нарезана по входам: Этап 4 «Калькуляторы» → вход А; Этап 2 «Бюджетный ремонт» → вход Б.

## Монетизация (по приоритету)
1. **Реф-комиссия с материалов** (и мебели) из сметы — и со СВОИХ ссылок юзера (deeplink),
   и с наших подборов. Чек материалов комнаты 50–200 тыс. ₽ × комиссия 1–5%.
2. **Мастера-лиды**: кнопка «найти мастера в городе» после сметы → партнёрская биржа (CPA).
3. **Paid-ступени** (позже, по данным): детальная смета/PDF/обновление цен; визуализация
   премиум. Цены — решение владельца.
4. Кэшбек юзеру как усилитель конверсии — опция, решение владельца (⏸).

## Этапы

### М0 — Партнёрки: LATE-BINDING (решение владельца 2026-07-11 — НЕ блокер запуска)
Схема: смета раздаётся сразу с обычными ссылками; **все внешние ссылки — только через наш
редирект `/go/<id>`** (резолв из БД в момент клика). Владелец ПОСТЕПЕННО регистрируется в
реф-программах магазинов (приоритет — по частоте магазинов в реальных сметах юзеров!);
после регистрации ссылка подменяется в БД → работает ретроактивно во всех сохранённых
чек-листах (юзер 100% вернётся — чек-лист для того и сохранял). Гдеслон пока НЕ
зарегистрирован (там есть все крупные магазины — кандидат №1); также прямые программы
магазинов и Admitad/ePN. При каждой регистрации проверять ToS на редиректы/подмену.
**Gate М0 снят** — реф-часть догоняет продукт, не блокирует его.

### М1 — Ядро «Смета-лист» (чек-лист навсегда)
Модель данных: смета / позиция (название, кол-во, формула, цена, url, ref_url,
источник: user_link|our_pick) / анонимная сессия (есть). **Обязательное: редирект-слой
`/go/<positionId>`** — единственный способ перехода наружу (условие late-binding М0);
клик логируется (какие магазины популярны → приоритет регистраций). Роутинг реф-программ —
**таблица маршрутов «домен магазина → программа → шаблон реф-URL»**: один магазин может быть
доступен через несколько сетей (Гдеслон / прямая программа / Admitad / ePN / AliExpress-портал) —
выбираем работающую с большей комиссией; смена программы = правка маршрута, чек-листы не трогаем. Страница сметы:
постоянная ссылка, список+сумма+«Мои комнаты»+шаринг. Позиция по ссылке: парс OG-меты,
деградация до «голая ссылка». v1.1 — ИИ-обогащение: сопутствующие с того же сайта
(агентный поиск), «подберём дешевле» где уверены. Цели: estimate_created / saved / ref_click.
**Gate: смета живёт по постоянной ссылке; клики идут через /go/ и логируются.**

### М2 — Вход А «Калькуляторы» (посадочные Этапа 4)
Формулы: обои (периметр×высота, раппорт, рулон), плитка (+10–15% подрезка), краска
(расход×слои), ламинат (+5–7%). Страницы `/calc/oboi|plitka|kraska|laminat` (SEO-friendly,
SSR): ввод размеров → ответ → «какие берёте?» → смета (М1). Фича «сфоткай — посчитаем»
(замер по фото из ml-наработок) — v1.5, после ручного ввода. **Gate: включаем кампанию
Этап 4 → kill-критерий уже задан (0 реф-кликов / 200 переходов → OFF, страница на SEO).**

### М3 — Вход Б «Сколько стоит ремонт» (посадочная Этапа 2)
`/calc/remont`: площадь|фото → глубина → стиль-свайп (есть) → вилка 3 бюджетов (прайс-база
категорий материалов из фидов/ручного прайса + грубые нормативы работ «отдельной строкой») →
смета (М1) с нашими позициями. Существующая кнопка лендинга «Рассчитать стоимость»
(/soon?f=cost) → сюда. **Gate: включаем Этап 2; CR клик→смета ≥ 8–10% на 200 кликов, иначе
чиним первый экран.**

### М4 — Реклама на полную + автопилот на все кампании
Автопилот: CAMPAIGN_ID → список всех кампаний (правка ~10 мин) + гейты. Этап 1 (AI-дизайн)
живёт параллельно — это теперь трафик хвоста «визуализация» и мебельного подбора.
Эшелон брендов и РСЯ — по прежней стратегии после валидации.

### М5 — Хвост «Визуализация + мебель» (бывшее ядро становится ступенью)
Из сметы: «посмотреть, как будет выглядеть» → существующий пайплайн (фото→стиль→превью)
+ мебельный подбор из фидов (sub-e2). Paywall переосмысляется под смету (детальная смета/
PDF/цены-обновление) — цены ⏸ владелец. Fit-точность («влезет/не влезет») — НЕ здесь,
отложена (архив).

### М6 — Мастера-лиды
Выбор партнёрской биржи (⏸ владелец), кнопка после сметы с гео, трекинг лидов. НЕ строим
свой каталог/рейтинг бригад.

### М7 — SEO и масштаб
Калькуляторы индексируются (вечный спрос ~90k/мес), контент «до/после», шаринг смет,
e-mail-возвраты (скелет sub-e7). Второй рынок (UK) — только после юнит-экономики RU.

## Механики, перенесённые из прежних планов (переписаны под смету; по указанию владельца)
- **М0.5 Concierge-валидация (из Э1/sub-e1, архив):** проверить деньгами ДО полного кода М1–М3 —
  первые сметы собирает владелец руками (заявка: размеры/ссылки → смета за 24ч, cap 10–20/нед,
  subid-атрибуция), fake-doors «Найти бригаду» и «Кэшбек» для замера интента. Гейты Э1
  переносятся: CPC ≤ 15–20 ₽ · клик→расчёт ≥ 20% · расчёт→сохранение ≥ 30% · смета→реф-клик
  ≥ 15% · kill: 0 реф-кликов/300 кликов → менять оффер/канал, не «ещё фича». (Утренние
  fake-doors Э1 «Посчитать материалы»/«Найти бригаду» подтверждают направление — переиспользовать.)
- **Безопасность (Э0/sub-e0, живой):** до трафика на новые страницы — лимиты трат и квоты
  генераций с $-kill-switch, «чужой UUID → 404», пин версий моделей.
- **Golden-eval формул (механика sub-e6, архив):** 10–20 эталонных комнат «размеры → правильные
  количества/суммы» как CI-тест; изменения формул/прайсов не проходят без зелёного golden.
- **Фиды (Э2/sub-e2, живой):** идемпотентный ресинк, ≥90% позиций с валидными полями,
  авто-подбор ≥70% CTR ручного; скоуп + категории МАТЕРИАЛОВ.
- **Правило доверия (владелец, 2026-07-11):** где релевантно — «подберём дешевле, если найдём»,
  даже если дешёвый магазин без нашей партнёрки: доверие и возвраты к чек-листу дороже
  разовой комиссии (возвраты монетизируются late-binding'ом позже).

## Метрики и kill-критерии
Воронка: визит → расчёт → смета создана → сохранена → реф-клик → (постбэк выкупа М0+).
Kill Этапа 4 задан; kill входа Б: CR клик→смета <5% после итераций первого экрана;
сквозной kill проекта (из брифа): CAC > 2× дохода после двух итераций — стоп, решение владельца.
