## Цель
Дать входу Б (вилка «сколько стоит ремонт») и «предвосхищению» реальные числа: база
стоимости РАБОТ по видам (₽/ед.) + доп. материалов, с региональной привязкой по IP+город.
Питает `MASTER-cost-first.md` М3 и обогащение чек-листа.

## Источник задачи
Владелец (2026-07-11): «база стоимости ремонта по регионам; медиана Москвы по каждому виду
работ (покраска, шпаклёвка, грунтовка…), от неё региональные коэффициенты по паритету цен;
регион юзера — по IP + спросить город; проверочно: сначала коэффициенты, потом точечная
сверка регионов. Предвосхищать: под ключевик давать не только ответ, но полный чек-лист
материалов/работ (сколько обоев → + клей, грунтовка, инструмент)».

## Стратегия данных (двухфазная — по владельцу)
**Фаза 1 — база Москвы + коэффициенты (MVP):**
1. Собрать медиану ₽/ед. по видам работ в Москве (веб-ресёрч: агрегаторы смет, прайсы бригад,
   Профи/Ремонтник, статьи-калькуляторы). Виды: демонтаж, грунтовка, шпаклёвка, штукатурка,
   покраска стен/потолка, поклейка обоев, укладка плитки, стяжка, ламинат/линолеум, электрика
   (точка), сантехника (точка), потолок (натяжной/ГКЛ) — единицы: ₽/м², ₽/пог.м, ₽/точку, ₽/шт.
2. Региональные коэффициенты к Москве (0.4–1.0): по индексу цен/доходов (Росстат, известные
   оценки «регион дешевле Москвы на X%»). Цена в регионе = медиана_Мск × k_регион.
3. Доп. материалы под каждый вид работ (нормо-расход + ориентир ₽): грунтовка л/м², клей,
   шпаклёвка кг/м², крестики/затирка и т.п. — «предвосхищение».
**Фаза 2 — верификация (позже):** точечно сверить 3–5 регионов реальными прайсами, скорректировать
коэффициенты. Не блокирует Фазу 1.

## «Предвосхищение по ключевикам» (маппинг)
Под каждый рекламный ключевик — что показать сверх прямого ответа. Таблица «ключевик →
прямой ответ + сопутствующий чек-лист». Примеры: «сколько нужно обоев» → рулоны + {клей,
грунтовка, шпатель, нож, отвес}; «сколько плитки» → м² + {клей, крестики, затирка, крестовина};
«сколько стоит покраска» → работа ₽/м² + {краска, грунтовка, валик, малярный скотч}.
Источник ключевиков — `domain/wordstat-semantics.md` (кластеры I/J + H).

## Регион пользователя
IP → регион (GeoIP; провайдер ⏸ — можно по заголовкам Caddy/Cloudflare или локальная база
MaxMind GeoLite2, бесплатная). Уточнение — спросить город на шаге расчёта (поле, дефолт из IP).
k_регион применяется к работам; материалы — цена федеральная (маркетплейсы ~едины).

## Модель данных (интерфейс для М3, чтобы код не ждал данные)
`lib/pricing/works.ts`: `WORK_RATES_MSK: Record<WorkType, {unit, rub}>` +
`REGION_K: Record<region, number>` + `rateFor(work, region)`. Наполнение — из ресёрча;
на старте — плейсхолдеры с пометкой `source: placeholder` до Фазы 1.

## Скоуп / НЕ входит
Входит: сбор медиан Москвы, коэффициенты, маппинг ключевик→чек-лист, GeoIP-схема, модуль
`lib/pricing`. НЕ входит: точные региональные прайсы (Фаза 2), интеграция с прайсами бригад
в реальном времени, юр-договоры со сметчиками.
