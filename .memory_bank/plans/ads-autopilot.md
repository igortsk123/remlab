## Цель
Автоматический контур защиты и оптимизации кампании Директ `712721026`: детект слива за минуты
(не сутки), авто-минусовка мусорных запросов, авто-добавление A/B-текстов при слабом CTR,
ежедневный отчёт в Telegram. Пайплайн «если…то» из независимых джобов на НАШЕМ сервере
с AI-судейством через Gemini. Мотивация владельца: у health-card ручной мониторинг раз в 1–2
суток пропустил сливы на ~10 000 ₽.

## Источник задачи
Владелец (2026-07-11): «джоб, который раз в период опрашивает Директ, анализирует сливы,
автоматом минусует, автоматом переписывает объявления где надо; пайплайн из промптов
"если…то"; согласен — пиши план большой, нет — возражай».

## Позиция: согласен, с 3 возражениями (безопасность денег и модерации)
1. **«Автоматом переписывать объявления» — НЕ заменой.** Робот только ДОБАВЛЯЕТ новый вариант
   в группу (A/B), работающие тексты не трогает и не выключает: авто-замена рискуетモдерацией,
   оффером и юр. формулировками. Отключение проигравшего — человеком на еженедельном разборе.
2. **Возобновление кампании после стоп-крана — только человек.** Авто-рестарт после авто-паузы
   = цикл пауза/старт и слепой слив. Пауза — автоматом, старт — вручную.
3. **Автономия ставок ограничена:** BidCeiling робот может поднять максимум до 25 ₽ (шаг 5 ₽,
   не чаще 1 р/48ч, только при «почти нет показов»); выше 25 ₽ — только человек.

## Ключевое архитектурное решение: НАШ сервер, не Remote Routine
Опыт health-card: их облачный джоб Клода «отработал, а отчёта не было ни разу» — **песочница
Anthropic блокирует egress к `api.direct.yandex.com` и `api.telegram.org`** (`host_not_allowed`;
их план `auto-minus-vps-cron.md` так и остался draft). Поэтому:
- **Хост:** exit-fi (наш прод-сервер), каталог `/opt/remlab/ads-watchdog/`, systemd-таймеры
  (прецеденты уже есть: `remlab-cleanup`, `remlab-watchdog`, `remlab-db-backup`).
  VPN-ноду не трогаем; джобы — лёгкие python-скрипты (curl-нагрузка нулевая).
  **Связность проверена с сервера 2026-07-11:** Директ API 200 (авторизованный запрос),
  Метрика 200, Telegram и Gemini доступны — зарубежный IP (Hetzner EU) Яндексу не мешает.
- **AI-судейство:** Gemini `gemini-flash-latest` (ключ УЖЕ на сервере в `/opt/remlab/.env`,
  цена — копейки: ~50 фраз × 12 ранов/день ≈ <$0.5/мес). Никаких новых подписок/слотов.
- **Уведомления:** Telegram-бот (⏸ владелец: токен нового бота из BotFather — рекомендую
  отдельного от health-card, изоляция проектов).
- **Код в git:** `infra/server/ads-watchdog/*` (python + unit-тесты решающих функций);
  на сервер — копированием при деплое (как cleanup.sh). Секреты — только в
  `/opt/remlab/ads-watchdog/.env` (chmod 600): `YANDEX_DIRECT_TOKEN`, `TG_BOT_TOKEN`,
  `TG_CHAT_ID`; Gemini — реюз из `/opt/remlab/.env`.

## Джобы и матрица «если…то»

### J-CHECK — стоп-кран и аномалии (таймер: каждые 30 мин; БЕЗ AI, чистые пороги)
| Если | То |
|---|---|
| State OFF→ON (баланс пополнен, показы пошли) | TG «🚀 кампания стартовала» + флаг `hot=48h` (учащённые проверки J-MINUS) |
| Расход сегодня > **650 ₽** (1.3 × дневной темп 500 ₽) | `campaigns.suspend` + TG «⛔ СТОП-КРАН: расход X ₽» + журнал |
| Расход за последний час > **300 ₽** | то же (слив соседей был 5 000 ₽/час — ловим на первой трети часа) |
| Средний CPC за день > 20 ₽ (потолок) | TG warn (аномалия стратегии) |
| Доля автотаргетинга в расходе дня > 30% | TG warn + пометка J-MINUS «AT-мусор, минусуй агрессивнее» |
| Status стал REJECTED (модерация отклонила) | TG alert со списком объявлений |
| Кампания SUSPENDED нами | no-op (ждём человека), напоминание в TG раз в сутки |
| Показов < 50/день два дня подряд при State=ON | J-BID: BidCeiling +5 ₽ (max 25 ₽, 1 р/48ч) + TG |

### J-MINUS — авто-минусовка (таймер: каждые 2 ч, 08:00–23:00 МСК; Gemini-судейство)
1. Reports API: `SEARCH_QUERY_PERFORMANCE_REPORT` за сегодня (offline-очередь с polling).
2. Новые запросы (не виденные ранее, state.json) → Gemini с критериями ниши:
   - **минусуем:** ванная/санузел/кухня-гарнитуры, ландшафт/участок/фасад, скачать/кряк/торрент,
     профи-САПР (автокад/3ds/рендер для дизайнеров), курсы/вакансии, игры/аниме, чужие
     бренды-магазины, «своими руками чертёж», б/у-объявления;
   - **НЕ минусуем (стоп-лист ядра, робот не может добавить):** нейросеть, ии, дизайн, интерьер,
     комната, квартира, фото, онлайн, бесплатно, планировщик, расстановка, мебель, ремонт, стиль;
   - принцип соседей: «сомневаешься — НЕ минусуй»; извлекать минимальный минус-корень.
3. Лимит **≤10 минусов/ран**; `campaigns.update` полным массивом; TG-отчёт «добавил/пропустил».

### J-ADS — тексты A/B (таймер: 1 р/сутки 10:00; Gemini-генерация; только ДОБАВЛЕНИЕ)
| Если | То |
|---|---|
| В группе ≥300 показов за 7 дн и CTR < 2.5% | Gemini генерит 1 новый вариант (правила: T1≤56/T2≤30/Text≤81, без «→» и спецсимволов, без чужих брендов, оффер «бесплатно/по фото/за минуты», без обещаний результата ремонта) → `ads.add` + `ads.moderate` + TG с текстом |
| В группе уже ≥4 объявлений | пропустить (не плодить), TG-заметка «нужен человек: отключить слабые» |
| Лимит | ≤1 новый текст на группу в неделю |

### J-REPORT — ежедневный отчёт (таймер: 21:00 МСК)
Сводка в TG: расход/клики/CPC/CTR по группам (Директ) + воронка целей 01→06 (Метрика API,
счётчик 110599064) + топ-5 поисковых запросов дня + действия роботов за сутки + рекомендации.
Раз в неделю (вс) — расширенный блок: сравнение групп, кандидаты на отключение, статус
эшелона 3/РСЯ.

### Kill-switch пайплайна
Файл `/opt/remlab/ads-watchdog/DISABLED` — существует → все джобы no-op («выключить всё» =
`touch DISABLED` или `systemctl disable --now remlab-ads-*`). TG-команды боту НЕ делаем (v1 —
только исходящие уведомления; двусторонний бот — отдельное решение позже).

## Пока кампания не крутится (сейчас: модерация, баланс пуст)
Джобы деплоим СРАЗУ, они дешёво «скучают»: J-CHECK ловит два события — «модерация прошла/
отклонена» и «показы пошли» (момент пополнения баланса) → TG. Т.е. владелец узнает о старте
рекламы автоматически, ручной проверки не нужно.

## Режимы внедрения (безопасность превыше)
1. **Э1 — каркас:** код в `infra/server/ads-watchdog/` (common.py: Direct/Metrika/Gemini/TG
   клиенты + state.json; jobs: check.py, minus.py, ads.py, report.py) + unit-тесты решающих
   функций (пороги, парс отчёта, валидация текстов) + systemd units/timers + logrotate +
   деплой-скрипт. Гейт remlab (pnpm) не трогаем; python-тесты — `python3 -m unittest` локально.
2. **Э2 — DRY-RUN (по умолчанию!):** 3–5 дней все джобы пишут в TG «что бы сделал», в Директ
   НЕ пишут ничего. Владелец смотрит адекватность решений.
3. **Э3 — боевой поэтапно:** сначала J-MINUS (низкий риск), +2 дня J-CHECK стоп-кран,
   затем J-BID, последним J-ADS.
4. **Э4 — наблюдение 2 недели,** тюнинг порогов, захват состояния в Memory Bank.

## Что остаётся человеку (не автоматизируем)
Возобновление после стоп-крана · отключение объявлений/групп/ключей · подъём потолков выше
25 ₽ · включение РСЯ и эшелона 3 · правки лендинга · решения по бюджету. Раз в неделю —
сессия «разбор рекламы» со мной: сверка действий роботов, захват в память (роботы Memory Bank
НЕ пишут — только TG и state.json; память обновляется на разборах).

## Риски
- Сервер общий с VPN-нодой → джобы лёгкие, без Docker, идут в obedient-каталоге /opt/remlab.
- Reports API офлайн-очередь бывает медленной → polling с backoff, таймаут 5 мин, ран пропускается.
- Gemini недоступен → J-MINUS/J-ADS скипают ран (деньги защищает детерминированный J-CHECK).
- Двойная правка минусов (робот+человек одновременно) → робот всегда читает свежий список
  перед записью и пишет полным массивом с merge.
