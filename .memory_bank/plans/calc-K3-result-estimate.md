## Цель
Экран результата с разбивкой по комнатам и стоимостью, агрегация по всему проекту, сохранение в
смету-лист (ядро M1) с /go/ реф. Здесь v2-билдер становится ОСНОВНЫМ (снимаем флаг) — на проде
появляется полноценный детальный калькулятор.

## Источник задачи
Роадмап `plans/calc-materials-roadmap.md` (К3), шаг 5 сценария + мост в смету.

## Скоуп — что входит
- **`lib/calc/result.ts`** — на комнату: gross/net площадь, кол-во материала, рулоны/плитки/упак/
  литры, запас, ориентировочная стоимость (по цене из параметров); агрегация по проекту (итоги).
- **`lib/calc/to-estimate.ts`** — материализация `CalcProject` → `Estimate`/`EstimateItem`
  (`contracts/estimate.ts`): позиции по комнатам/материалам + сопутствующие (`companions.ts`),
  `source:"calc"`, `url`→`domain` для /go/. Можно ДОПОЛНЯТЬ существующую смету.
- **`components/calc/ResultView.tsx`** — экран «Рассчитать»: итоги, разбивка по комнатам,
  сопутствующие «не забыть», стоимость, кнопка «Сохранить смету». loading/empty/error.
- **`app/calc-actions.ts`** (или расширить `app/estimate-actions.ts`) — server action: сохранить
  результат в `Estimate` (repository), `getSessionId`, redirect на `/e/[id]`; событие `estimate_created/saved`.
- **`app/calc/[kind]/page.tsx`** — ПРОМОУТ: v2-билдер по умолчанию (снять флаг-гейт); прежний
  `CalcForm` вывести из основного пути (файл оставить/удалить по чистоте).
- Аналитика: воронка `calc_started → calc_result → estimate_saved`.
- Тесты: `lib/calc/result.ts` (итоги) + `to-estimate` (маппинг позиций).
