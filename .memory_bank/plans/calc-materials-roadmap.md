# МАСТЕР-РОАДМАП — Калькулятор материалов v2

> Верхнеуровневый план. Каждая фаза К0–К6 = отдельный подплан (`draft` → «деплой» →
> `completed`) по `.claude/rules/agent-workflow.md`, выполняются ПО ОЧЕРЕДИ. Этот файл — карта,
> его самого не «деплоим»; «деплоим» подпланы.

## Цель
Достроить «Вход А — калькуляторы» до детального инструмента из сценария владельца: выбор
материала → мультикомната → детальная геометрия с авто-вычетом проёмов → полные параметры
материала (ручные или автозаполнение по ссылке) → результат с разбивкой по комнатам и
стоимостью → платные хвосты (визуализация 60 ₽, «найти дешевле»). Итог по-прежнему втекает в
смету-лист (ядро M1) с /go/ реф-ссылками.

## Источник задачи (сценарий владельца, 6 шагов)
1. Выбор типа расчёта: обои / плитка / краска / ламинат. Данные — локально в текущей сессии.
2. Мультикомната: по умолчанию «Комната 1»; добавить/удалить/редактировать комнаты.
3. Геометрия. Обои/плитка/краска: стены/поверхности — длина, высота, окна, двери, прочие
   исключаемые проёмы (автоматически вычитаются из площади). Ламинат: длина+ширина пола,
   доп. зоны, исключаемые зоны (каждая — длина×ширина).
4. Параметры материала — вручную ИЛИ вставить ссылку на товар → система находит характеристики в
   интернете и заполняет поля.
   - **Обои:** ширина рулона, длина рулона, раппорт, укладка со смещением/без, цена/рулон, запас.
   - **Плитка:** длина, ширина, ширина шва (дефолт 3 мм, меняется), цена за м²/упак, шт/упак, запас на подрезку.
   - **Краска:** тип поверхности (штукатурка/шпаклёвка/дерево/металл/др), тип краски, расход
     (авто по типу или вручную), число слоёв, объём упаковки, цена упаковки.
   - **Ламинат:** длина панели, ширина панели, толщина, шт/упак, цена за м²/упак, смещение рядов
     (произвольно / 1⁄2 / 1⁄3), мин. длина обрезка, отступ от стен (8–10 мм, дефолт 10),
     направление укладки (по длине / по ширине / диагональ 45° / диагональ 135°).
5. Результат («Рассчитать»): общая площадь; площадь без проёмов/исключений; нужное кол-во
   материала; рулоны/плитки/упаковки/литры; рекомендуемый запас; ориентировочная стоимость;
   разбивка по каждой комнате.
6. Доп. услуги: **Визуализация 60 ₽** (фото комнаты → как будет с выбранными материалами);
   **«Найти дешевле»** (есть ссылка → авто; нет → добавить; поиск аналога/скидок; результат за
   e-mail / подписку на Telegram-бота / подписку на бота в MAX).

## Связь с мастер-планом (MASTER-cost-first)
- **Ядро роадмапа = глубокая достройка M2 «Вход А калькуляторы».** Текущий M2 — облегчённый
  (одна комната, одна поверхность); здесь — полноценный многокомнатный расчётчик.
- **Хвосты пересекаются с другими ступенями:** визуализация 60 ₽ → механика M5 (пайплайн `/start`
  `/p/*` + оплата); «найти дешевле» + лид-магнит (email/бот) → M6/M7 (рост/лиды/SEO-возвраты).
- **Мост в M1:** результат материализуется в `Estimate` (смета-лист) с /go/ реф — не дублируем ядро.
- Ядро сметы M1 НЕ переписываем; калькулятор — источник позиций.

## Ключевые архитектурные решения (общие для подпланов; финализируются в К0)
1. **Рабочее состояние — клиентское, локально в сессии** (как в сценарии): комнаты/поверхности/
   параметры держим в клиентском состоянии + `localStorage` (переживает перезагрузку вкладки).
   Сервер/БД трогаем ТОЛЬКО когда пользователь «Сохранить смету» → materialize в `Estimate` (M1).
2. **Формулы — чистые функции** в `lib/estimate/calc.ts` (+ новые модули), покрыты golden-eval
   (эталонные комнаты «размеры→кол-во/сумма» как CI-тест; правки формул не проходят без зелёного).
3. **Контракты калькулятора (Zod)** отдельно от `Estimate`: `CalcProject { rooms[] }`,
   `Room { kind, surfaces[]|floor, materialSpec }`, `Surface/Zone`, `MaterialSpec` (юнион по виду).
   Тип — из Zod (`z.infer`). Валидация на границах (формы/парсер ссылки).
4. **Автозаполнение по ссылке** — сервисный слой с деградацией: не удалось распарсить → ручной ввод
   (маркетплейсы режут ботов, см. Риски). Ссылка → позиция сметы с /go/ реф.
5. **Платные/лид-функции — отдельные ступени** (К5/К6): оплата, ПДн, боты — по решению владельца,
   юр. логику не реализуем (TODO по CLAUDE.md).

## Дорожная карта — фазы (каждая → отдельный подплан draft→деплой)

### К0 — Каркас: контракты + локальное состояние + мультикомната
- Zod-контракты калькулятора (CalcProject/Room/Surface/Zone/MaterialSpec).
- Клиентское состояние + `localStorage`; выбор вида → «Комната 1» по умолчанию; добавить/удалить/
  редактировать/переименовать комнату; переключение вида на комнату.
- Каркас страницы-билдера (шаги), пустые состояния. Формул ещё нет.
- **Gate:** можно завести 2–3 комнаты, данные переживают перезагрузку; типы/линт/тест зелёные.

### К1 — Геометрия и площади (проёмы и зоны)
- Обои/плитка/краска: добавление стен/поверхностей (длина, высота) + окна/двери/прочие исключения
  → авто-вычет из площади. Ламинат: пол (длина×ширина) + доп. зоны + исключаемые зоны.
- Чистые функции площади: gross (общая) и net (без проёмов/исключений). Golden-тесты.
- UI ввода поверхностей/зон (добавить/удалить, несколько на комнату).
- **Gate:** корректная net/gross площадь на эталонных комнатах (golden); проёмы вычитаются.

### К2 — Параметры материалов (ручной ввод) + полные формулы
- Полные параметры и формулы по 4 видам (обои: раппорт + смещение; плитка: шов 3 мм + подрезка +
  шт/упак; краска: тип поверхности/краски + расход авто/ручной + слои + объём; ламинат: смещение
  1⁄2·1⁄3·произв + отступ 10 мм + направление вкл. диагональ + мин. обрезок).
- Умные дефолты (шов 3 мм, отступ 10 мм, расход по типу краски, запас по виду/укладке).
- Расширить `lib/estimate/calc.ts` (+ модули); golden-eval на каждую формулу.
- **Gate:** для каждого вида ручной ввод → корректное кол-во с запасом (golden зелёный).

### К3 — Результат + агрегация по комнатам + мост в смету
- Экран «Рассчитать»: общая/чистая площадь, кол-во материала, рулоны/плитки/упак/литры, запас,
  ориентировочная стоимость, разбивка по каждой комнате; сопутствующие (companions).
- Агрегация по всем комнатам проекта.
- «Сохранить смету» → materialize в `Estimate` (M1): позиции + цена + /go/ реф; открывается по
  постоянной ссылке (`/e/[id]`), доступно в «Моей лаборатории».
- **Gate:** воронка calc_started → расчёт → смета сохранена; итоги сходятся с golden.

### К4 — Автозаполнение по ссылке на товар
- Вставил ссылку → сервис тянет характеристики (OG-мета/парсер/поиск) → заполняет поля материала
  выбранного вида; редактируемо.
- **Деградация:** не удалось → «голая ссылка + ручной ввод» (чек-лист ценен и так).
- Ссылка → позиция сметы с /go/ реф (какие магазины популярны → приоритет реф-регистраций М0).
- **Gate:** для ≥1 доступного источника автозаполнение работает; при неудаче — чистый fallback.

### К5 — Визуализация за 60 ₽
- Из результата: загрузить фото → комната с выбранными материалами (обои/краска/плитка/ламинат).
- Переиспользовать пайплайн `/start` `/p/*` (restyle по эталону, ступень M5) + трейсинг.
- **Оплата 60 ₽** — интеграция YooKassa (сейчас заглушка); честный дисклеймер «концепт».
- **Решения владельца:** цена (60 ₽ дана), провайдер оплаты, ПДн — юр. часть TODO, не блокируемся.
- **Gate:** оплата проходит (тест-режим) → генерация с выбранными материалами → результат.

### К6 — «Найти дешевле» + лид-магнит
- Кнопка «Найти дешевле»: есть ссылка → авто; нет → добавить вручную. Поиск аналога/скидок/акций
  (правило доверия: «подберём дешевле, даже без нашей партнёрки» — возвраты дороже разовой комиссии).
- Гейт результата: e-mail / подписка на Telegram-бота / подписка на бота в MAX.
- **Новое:** интеграции Telegram-бота и MAX-бота; хранение лида.
- **Решения владельца / юр.:** ПДн (152-ФЗ) при e-mail/подписке — согласие/политика (TODO/юрист);
  выбор каналов; что реально ищем «дешевле» и по каким источникам.
- **Gate:** лид фиксируется; подписка/►email открывает результат; ПДн-TODO зафиксирован.

## Решения владельца / открытые вопросы (уточняем по мере подпланов)
1. Цена визуализации — **60 ₽ (дана)**; провайдер оплаты — YooKassa (стек); подтвердить.
2. Источники автозаполнения по ссылке (какие магазины в приоритете; ToS на парсинг).
3. «Найти дешевле» — по каким источникам ищем аналоги/скидки; глубина в MVP.
4. Лид-магнит: какие каналы обязательны (email и/или Telegram и/или MAX); MAX-бот — новая интеграция.
5. ПДн/152-ФЗ при сборе email и подписках — юр. оформление (TODO по CLAUDE.md, не блокирует код).
6. Дефолты запасов/расходов (подтвердить нормативы: подрезка плитки 10/15%, ламинат 5–7%, расход красок по типам).

## Риски и обходы
- **Парсинг маркетплейсов (К4):** Ozon/WB/Леруа режут ботов → автозаполнение может не сработать.
  Обход: деградация к ручному вводу; позже — данные из фидов сетей (sub-e2). Не блокер.
- **Точность формул (К1–К2):** ошибки в кол-ве бьют по доверию. Обход: golden-eval эталонных комнат
  как CI-гейт; честный дисклеймер «проверьте перед покупкой».
- **Оплата и боты (К5–К6):** новые интеграции (YooKassa, Telegram, MAX). Обход: отдельные ступени,
  тест-режимы, откатываемость; не тянуть в ядро калькулятора.
- **Разрастание состояния (мультикомната × поверхности × параметры):** держать модель плоской и
  типобезопасной (Zod-юнионы по виду), localStorage-схема версионируется.

## Что НЕ входит (scope роадмапа)

- Полноценный маркетплейс-парсер под все площадки (в К4 — MVP + деградация).
- Юридическая логика ПДн/лицензий (TODO, не реализуем — CLAUDE.md).
