## Цель
Превратить ценовой спрос («сколько стоит ремонт» ~52k, калькуляторы материалов ~70–90k/мес)
во вход воронки с монетизацией через реферальную смету. Два флоу от владельца (2026-07-11):
«знает, что хочет» → смета из ЕГО ссылок с нашей комиссией; «не знает» → AI-расчёт под
стиль и бюджет. Это продуктовая начинка уже залитых кампаний Этап 2/4 и эшелона 3.

## Источник задачи
Владелец (голосом, 2026-07-11): «при запросе "сколько стоит ремонт" уводить в расчёт;
спросить, есть ли конкретные ссылки — превращать их в реферальные и отдавать ему же;
сохранить как страницу со всем списком, чтобы ничего не забыть; комиссия с каждого товара,
который он сам подобрал. Второй случай: не знает, что хочет — напиши/скажи/загрузи фото,
площадь → стили → подберём ремонт под разный бюджет».

## Флоу 1 — «Своя смета» (человек ЗНАЕТ, что хочет)
1. Страница «Смета ремонта/материалов»: юзер вставляет ссылки на товары, которые сам выбрал
   (Ozon/WB/Леруа/Петрович…), и/или выбирает категории (обои, плитка, краска, ламинат).
2. Мы считаем КОЛИЧЕСТВА (наш дифференциатор): по площади/фото комнаты (замер по фото уже
   есть в ядре) — рулоны обоев (периметр×высота), плитка +10% запас, краска (расход×слои).
3. Ссылки конвертируем в реферальные (deeplink CPA-сети) и собираем «страницу-смету»:
   список позиций, количества, сумма, «ничего не забыть» — сохраняется в «Мои комнаты»
   (анонимная сессия уже есть).
4. Он покупает по своим же ссылкам → нам комиссия. По сути кэшбек-механика без кэшбека;
   опция-усилитель (⏸ владелец): отдавать часть комиссии юзеру как кэшбек → выше конверсия.

**Почему сильно:** интент момента покупки; человек кликает по товару, который сам выбрал →
конверсия клик→покупка максимальна; MVP почти без AI-затрат.

## Флоу 2 — «AI-расчёт» (человек НЕ знает, что хочет)
Вход «Рассчитай стоимость ремонта»: свободный текст (v2 — голос) + опц. фото + площадь →
стили (свайп уже есть) → 2–3 варианта под РАЗНЫЙ бюджет (эконом/средний/выше) со сметой и
реф-товарами. Технически = существующий пайплайн (бриф→фото→стиль→подбор→budget в превью)
+ Cost Engine (ступень 3 v0.3, в коде НЕТ) + бюджетные вилки. Кнопка «Рассчитать стоимость»
УЖЕ на лендинге — сейчас ведёт в заглушку `/soon?f=cost` → станет входом Флоу 2.

## Ключевые вопросы выполнимости (проверить ДО кода)
1. **Гдеслон deeplink** (критично для Флоу 1): умеет ли произвольную ссылку → партнёрскую;
   какие DIY-магазины в сети (Леруа? Петрович? Ozon/WB — свои программы с ограничениями);
   комиссии на стройматериалы (ждём 1–5%). Нужен аккаунт Гдеслон (⏸ владелец, из v0.3).
   Fallback-сети: Admitad/ePN. Если магазин юзера не в сети — показываем ссылку как есть
   (ценность списка сохраняется) + предлагаем наш аналог из фидов.
2. **Экономика материалов:** чек ниже мебельного; комиссия мала → ставка на объём (70–90k
   спроса) и на до-продажу «как это будет выглядеть» (генерация) + мебель рядом.
3. **ToS сетей:** покупка юзером по реф-ссылке, которую сгенерил сервис — стандартная
   кэшбек-механика (LetyShops и др.), нарушения нет; перепроверить правила Гдеслон.
4. CPC кластера J — «Прогнозом бюджета» перед включением Этапа 4 (уже в плане ads-bath-calc).

## MVP-срез (когда владелец скажет «делаем»)
- Флоу 1: страница списка-сметы (ссылки + количества по площади) + сохранение + реф-конверсия
  для магазинов из сети; без каталога, без кэшбека. Посадочная Этапа 4 = эта страница
  (её kill-критерий уже задан: 0 реф-кликов / 200 переходов → OFF, страница на SEO).
- Флоу 2: `/soon?f=cost` → реальный вход: бриф с площадью → существующий флоу + вилка
  бюджета (поле budget уже есть) — БЕЗ Cost Engine в MVP (честная «оценка по подбору»).
- Голос, кэшбек, Cost Engine полноценный — v2+.
