## Цель
Поднять точность **замера зоны и объектов** (ради надёжного «влезет/не влезет»), заменив слабые CPU-звенья на
точные **GPU-модели через fal.ai** (pay-per-call, $0 в простое). Архитектуру не переписываем — меняем модели
в существующих слотах. Цель точности: **±10%** на пол/зонах, чистые маски для выреза.

## Источник задачи
Владелец (2026-07-06): «раз GPU есть по API — можно точнее всё выявлять; докрути точность замера зоны
(SAM2 + Depth через fal), чтобы влезет/не влезет был надёжнее по текущим объектам». Цели — `../goals-one-photo-furnish-fit.md`.

## Что меняем по звеньям (модель → выигрыш)
| Звено | Сейчас (CPU) | Меняем на (fal GPU) | Выигрыш |
|-------|--------------|---------------------|---------|
| **Маски объектов** | Mask R-CNN (грубые края) | **SAM 2** (бокс→точная маска) | пиксель-точный footprint зоны + чистый вырез. **Главный выигрыш для fit-check** |
| **Глубина** | Depth Anything V2 **Small** | **Depth Anything V2 Large / Metric3D** | точнее 3D → площади/высоты/зоны точнее |
| **Детекция** | Mask R-CNN (COCO-классы) | **+ Grounding DINO** (open-vocab) | добрать радиатор/шторы/двери/зеркала |
| **Окна/стекло** | по маске (тянется низко) | **маска стекла** (GDNet-класс) → окно по раме/стене | подоконник/окно без «глубины улицы» |

## Что ОСТАЁТСЯ на CPU (бесплатно, точно)
- Детект A4 + `solvePnP` (масштаб/поза — сверено 0.3%, GPU не нужен).
- Оркестрация, вся геометрия, fit-логика, рендер — наш Python (fal зовём по HTTP).

## Как замер ЗОНЫ станет надёжнее (связь с «влезет/не влезет»)
1. SAM2 → точная нижняя кромка объекта → точная проекция на пол → **footprint зоны ±5%** (сейчас грубо).
2. Depth Large → площади/свободный пол точнее.
3. Итог: «влезет/не влезет» держит ±10%, пограничные метим «впритык».

## ПЕРЕПРИОРИТИЗАЦИЯ (owner 2026-07-06): сначала АЛГОРИТМ, потом правка человеком
Владелец: «правка циферок человеком — задача вторая, первая — алго улучшать, поэтапно». И три замечания по room2
(монитор, вещи на столе, увлажнитель в углу) — это ОДИН пробел: детекторы знают ~80 COCO-классов, не знают
увлажнитель/монитор/мелочь. Чинить не перечислением, а **open-vocab + иерархией «что на чём»**.
Порядок фаз алгоритма: **Ф1 (видеть всё) → Ф2 (иерархия что-на-чём) → Ф3 (само-проверяющий замер)**.

- [ ] **Ф1. Видеть ВСЁ (Grounding DINO через fal)** — open-vocab словарь (monitor, humidifier, fan, lamp, plant,
  box, books, bottle, router, speaker…) + generic «object on table/floor». Ловит незнакомые предметы (увлажнитель,
  монитор, мелочь на столе). Каждый предмет = бокс+ярлык, даже неизвестный. Затем SAM2 уточняет маску.
- [ ] **Ф2. Иерархия «что на чём» (containment)** — геометрически: footprint предмета над верхней плоскостью
  другого → «на столе»/«на матрасе»; иначе «на полу». Вещь на столе мерить от столешницы, НЕ от пола.
- [ ] **Ф3. Само-проверяющий замер** — каждый размер 2-3 методами → сверка с таблицей типовых → не сходится →
  другой алго → confidence/provenance. (Дизайн движка — в `../goals-one-photo-furnish-fit.md`.)
- [ ] **Ф4. Depth Large/Metric3D через fal** — пересчитать размеры/площади → сверить с эталонами (стол 200, потолок 263, стул 108).
- [ ] **Ф5. Маска стекла** — окна/подоконник по раме+стене, не по глубине.
- [ ] **Ф-UX (ПОТОМ). Правка человеком** — фото с выделенными объектами + поля Ш/В/Г, confidence-флаги ведут глаз.
