## Цель
В липком итоге сверху калькулятора показать суммарное КОЛИЧЕСТВО материала (для плитки — штук)
между строкой площади («X м²») и ценой («≈ Y ₽»).

## Источник задачи
Промпт владельца: «в итоге добавь количество посчитанной плитки после метров и перед ценой».
Речь про липкий итог `calc-sticky` в `CalcBuilder.tsx` (Площадь → [сюда количество] → цена).

## Скоуп — что входит
- В `CalcBuilder.tsx`: посчитать суммарное количество по всем частям всех комнат (исключая части
  без известного размера — `qtyUnknown`), с единицей вида (плитка → шт, обои → рулон, краска → л,
  ламинат → упаковка). Показать строкой «Нужно: N <ед.>» МЕЖДУ площадью и ценой в `.calc-sticky`.
- Импорт `pluralUnit` для склонения единицы.

## Скоуп — что НЕ входит
- Логика формул/контрактов — не трогаем (берём готовые `out.qty`/`out.unit`).
- Блок `ResultView` (там количество по строкам уже есть) — не трогаем.
- Части без размера плитки (`qtyUnknown`) в сумму штук не входят (штуки неизвестны).

## Файлы к изменению
