## Цель
На `/calc/plitka`: (1) считать плитку для пола отдельной плиткой (стены и пол — РАЗНЫЕ плитки, две
строки итога), (2) уменьшить кнопку «Заполнить», (3) починить парсинг цены/размера с тяжёлых страниц.

## Источник задачи
Промпт владельца по `/calc/plitka` + уточнение: пол считать СВОЕЙ плиткой, отдельно от стен.
Крупная кнопка «Заполнить». Ссылка `stroyparkdiy.ru/good/15215301` — не подгрузилась цена.

## Ключевой контекст (разобрано)
- **Модель (ADR-0018):** проект = один `kind`; комната имеет `material` (спека), `surfaces` (стены),
  опц. `floor`, `productUrl`. Для плитки сейчас считаются только стены; `floor`/`FloorEditor` — у ламината.
  Мультикомната + пер-комнатный `material` + суммарный Итог уже есть (К0–К3).
- **Раздельные плитки:** в одну комнату добавляем ВТОРУЮ спеку — для пола. Комната для плитки может
  дать ДВЕ позиции: стены (по `material`) и пол (по новому `floorMaterial`). Каждая — со своей ссылкой
  и параметрами. Абстракция `computeRoomParts(room,kind)` → массив частей {label, out, material, url}.
- **#3 цена:** stroypark = **1.26 МБ**, а `parse-link` режет HTML до **500 КБ** → цена (позиции
  715–760 КБ) отсекается. Главный ценник `itemprop="price" content="2 220.00"` (класс
  `carrot-product-price`, первый на странице); регэксп `[\d.,]+` рвётся на пробеле тысяч. og:title
  «20х20**см**» → сейчас 20×20 **мм** (×10 меньше).

## Скоуп — что входит
1. **Пол со своей плиткой (стены+пол раздельно):**
   - контракт `room`: `floorMaterial?` (спека пола) + `floorProductUrl?` (ссылка пола);
   - `computeRoomParts(room,kind)`: для плитки → части «Стены» (если есть стены) и «Пол» (если задан
     `floor`), каждая считается своей спекой через `tile()`; для остальных видов → одна часть = `computeRoom`;
   - `RoomPanel` (плитка): секция «Стены» (как сейчас: `SurfaceEditor` + `LinkAutofill`+`MaterialParams`
     на `material`/`productUrl`); кнопка «+ добавить размеры пола» → секция «Пол» (`FloorEditor` +
     ВТОРОЙ `LinkAutofill`+`MaterialParams` на `floorMaterial`/`floorProductUrl`) + «удалить пол»;
     результат-note показывает обе части;
   - `ResultView` (Итог) и `lib/calc/to-estimate.ts`: строка/позиция на КАЖДУЮ часть
     («{комната} — стены», «{комната} — пол»); суммарная стоимость складывает части.
2. **Кнопка «Заполнить»** (`LinkAutofill`) — мельче (padding/шрифт).
3. **Парсинг крупных страниц:** (а) лимит HTML в `parse-link` 500 КБ → 2 МБ; (б) цена — допустить пробел
   в числе («2 220.00» → 2220); (в) размеры плитки/ламината в «см» → в мм (×10).

## Наименование (не привязываться к «ванной»)
Фича — для ЛЮБОГО помещения (коридор, кухня, прихожая, санузел…). Имя комнаты — пользовательское
(дефолт «Комната N», НЕ «Ванная»). Части подписаны нейтрально: «Стены» / «Пол». Никаких хардкодов
типа помещения в UI/копирайте.

## Скоуп — что НЕ входит
- Разные плитки в пределах ОДНОЙ поверхности (напр. две плитки на одной стене) — нет.
- Авто-«шт/упаковка» для stroypark (на странице нет явного «N шт в упаковке», только «1 м²») — ручной ввод (follow-up).
- Формула `tile()` и её golden-тест — не трогаем (меняем только откуда берём площадь/спеку).
