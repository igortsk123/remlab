## Цель
Второй проход UX/копирайта калькулятора обоев + починка автозаполнения размеров рулона из ссылки
и расширение лид-формы «Найдём выгоднее» (мультитовар + город).

## Источник задачи
Промпт владельца по `/calc/oboi` (8 пунктов), включая реальную ссылку с непрочитанными размерами:
`miroboev-shop.ru/.../r210139-...-1-06-10m-...` (рулон 1,06×10 м).

## Ключевой контекст (разобрано)
- **#5 парсинг:** на странице miroboev есть ВСЕ три значения, но парсер их не берёт:
  - размеры рулона — в `og:title`/`og:description` как «**1,06*10м**» (разделитель `*`, десятичная `,`,
    без слов «ширина/длина»). Текущий парсер ждёт слова «ширин[аы]/длин[аы]» или целые через `x/×` → мимо.
  - раппорт — в таблице характеристик body: `<td>Рапорт смещения рисунка:</td><td><span>64 см</span></td>`.
    Нюансы: «**Рапорт**» с ОДНОЙ «п» (не «раппорт»); значение в СОСЕДНЕМ `<td>` (между ними теги) →
    парсить по тексту со СНЯТЫМИ тегами; см→м. Проверено симуляцией: `рап+орт[^0-9]{0,40}(\d…)(см|мм|м)`
    матчит «Рапорт смещения рисунка: 64 см» → 0.64 м, и идёт в документе ПЕРЕД пустым полем-лейблом
    их калькулятора «раппорт (см)» → ложного срабатывания нет.
  - Ловушка: «Ширина/Длина комнаты (м)» на странице — поля ИХ калькулятора, НЕ рулон (word-based подход опасен).
  `MaterialParams` уже показывает «Ширина/Длина рулона, м» + «Раппорт, м» — фикс целиком в парсере.
- **#7 лид-форма:** `captureLead` → `leadRepo` → таблица `leads` (id/email/channel/url/kind/session).
  Нужны мультиссылки + город → добавить колонку `city` (идемпотентный ALTER в `005-leads.sql`,
  он уже в `deploy.sh` 5b), мультиссылки хранить joined в `url`.
- Тексты по видам — через `CALC_META` (падежи), чтобы плитка/краска/ламинат оставались грамотными.

## Скоуп — что входит
1. **«+ добавить размеры стены»** — плюс перед текстом кнопки (`SurfaceEditor`, все виды).
2. **Иконка проёмов → дверь**: прямоугольник + точка-ручка (вместо «окна»-крестовины).
3. **«убрать» → «удалить»** на стене + сделать чуть мельче (шрифт).
4. **Текст подсказки** (tooltip) → «Окна и двери не вычитаются из расчёта, т.к. обои клеятся целыми
   полосами, поэтому кусок, оставшийся из-за проёма, обычно нельзя использовать в другом месте. За счёт
   этого получается небольшой запас: на подгонку рисунка, обрезки и непредвиденные ошибки.»
5. **Парсинг ссылки (обои):** (а) формат «W×L м» (`1,06*10м`, `0,53х10,05 м`, `1.06 x 10 м`) →
   `rollWidthM`/`rollLengthM` (guard: W<2.5, L∈3..50), из meta (og:title/description); (б) раппорт —
   `рап+орт` (1–2 «п») + ЧИСЛО + единица (см/мм/м) по body со СНЯТЫМИ тегами → `rapportM` в метрах
   (64 см → 0.64), guard 0<r<2; пустой лейбл «раппорт (см)» без числа НЕ матчить. Деградация к ручному вводу сохраняется.
6. **Текст «Найдём выгоднее»** → «Мы найдём {ваши обои} и все необходимые материалы для {поклейки} по
   более выгодной цене в магазинах вашего города или онлайн.» (по видам через новый `accYours`).
7. **Раскрытая форма «найти выгоднее»:** поле ссылки на товар → ниже «+ добавить товар» (мультиссылки)
   → поле «Ваш город» → далее как есть (e-mail, согласие, «Прислать варианты», боты).
8. **Кнопка визуализации** → «Посмотреть комнату в новом дизайне» (было «…в дизайне по фото»).
