## Цель
Ввести **сквозную шапку** (единое верхнее меню на всех страницах) со сценариями-разделами и
двумя выделенными кнопками-калькуляторами; пересобрать **главную** в «рекламу проекта целиком»
(не только калькулятор); завести каркасы новых разделов: **Стили** (игра «узнай свой вкус» +
статьи-плейсхолдеры), **Советы** (плейсхолдеры), **Моя лаборатория** (хаб сохранённого).

## Источник задачи
Промпт владельца (2026-07-12): наверху нужны разные сценарии — калькулятор, примеры дизайна
(фото→дизайн), статьи со стилями (карточки-игра «определи вкус» + чтение про стили, SEO),
личный кабинет (позже), советы по ремонту «своими руками». Две кнопки «Посчитать материалы» и
«Сколько стоит ремонт» сделать сквозными в шапке. Главную — на усмотрение (общая реклама
проекта: «не только калькулятор, а помощь сделать ремонт своими руками — хорошо и недорого»).

Уточнения (AskUserQuestion, 2026-07-12):
1. Шапка: **все разделы на виду**, две основные кнопки выделить шрифтом/подсветкой (не «гамбургер»).
2. Стили/Советы: **сейчас без наполнения — плейсхолдеры**, тексты позже.
3. Дизайн-раздел в этот заход: **игра-карточки «узнай свой вкус»** + статьи по стилям (позже).
4. Кабинет: назвать интересно — **«Моя лаборатория»** (на бренд remont-lab), не «Мои сметы».

## Скоуп — что входит
1. **Сквозная шапка** `components/SiteHeader.tsx` (client, `usePathname` — подсветка активного
   раздела), монтируется в `app/layout.tsx` над `{children}`. Sticky сверху, семантические токены.
   - Строка 1: логотип `remont-lab` → `/` (слева), «Моя лаборатория» → `/lab` (справа).
   - Строка 2 (навигация): **[Посчитать материалы]** `/calc` и **[Сколько стоит ремонт]**
     `/calc/remont` — выделенные кнопки-пилюли; далее лёгкие текст-ссылки: **Дизайн** `/start` ·
     **Стили** `/styles` · **Советы** `/sovety`.
   - Мобайл: логотип-строка + горизонтально-скроллящаяся полоса навигации (всё на виду, без
     «гамбургера»); две кнопки — залитые пилюли в начале полосы, остальное — ghost-ссылки.
     Контроль: страница не уезжает по горизонтали (`overflow-x` только внутри полосы).
2. **Главная** `app/page.tsx` — пересборка в «о проекте в комплексе»:
   - Герой: «Ремонт своими руками — хорошо и недорого» + подзаголовок, что умеет remont-lab.
   - Две основные CTA-кнопки (калькуляторы).
   - Блок-грид «Что внутри» — карточки широты проекта: Посчитать материалы · Сколько стоит ремонт ·
     Дизайн по фото · Узнай свой стиль · Советы по ремонту · Моя лаборатория (сигнал «не только
     калькулятор»).
   - «Как это работает» (сохранить/освежить существующие шаги) + хвост «дизайн по фото».
3. **Новый раздел «Стили»** `app/styles/page.tsx`:
   - Наверху — **игра-карточки «Узнай свой вкус»** `components/StyleQuiz.tsx` (client): показываем
     карточки интерьеров по одной → «Нравится» / «Не моё» → после N карточек считаем
     доминирующий стиль → экран результата «Ваш стиль — …» + короткое описание + CTA:
     «Посмотреть свою комнату в этом стиле» (`/start`), «Собрать смету» (`/calc`), «Читать про
     стиль» (плейсхолдер). Данные-сид: `lib/styles/quiz.ts` (стиль, подпись, картинка).
     ⚠️ Реальные фото интерьеров — content-later (Q2): сейчас **карточки-плейсхолдеры**
     (стилизованная плитка: название стиля + палитра/градиент из токенов), механика игры полная.
   - Ниже — список статей по стилям как **плейсхолдеры** («скоро», без текстов).
4. **Новый раздел «Советы»** `app/sovety/page.tsx` — плитки-категории (Стены, Пол, Потолок,
   Поклейка обоев, Грунтовка, Штукатурка) как **плейсхолдеры** («скоро»); каркас под будущие статьи.
5. **«Моя лаборатория»** `app/lab/page.tsx` — хаб сохранённого: карточки «Мои сметы»
   (→ `/estimates`) и «Мои дизайны/комнаты» (→ `/rooms`) со счётчиками и пустыми состояниями.
   Существующие `/estimates` и `/rooms` остаются, шапка ведёт на `/lab`.
6. **Гигиена бренда:** убрать дублирующий `<p className="eyebrow">remont-lab</p>` со страниц, где
   он был верхним элементом (грепнуть; бренд теперь в шапке).
7. **CSS** `app/globals.css` — стили `.site-header`/навигации/выделенных кнопок + карточки квиза;
   только семантические токены (ui-rules), mobile-first.
8. **Лёгкая аналитика** (`lib/analytics.ts`): событие завершения квиза (`quiz_completed` + стиль) —
   новая мини-воронка; no-op без ключа PostHog.

## Скоуп — что НЕ входит
- Реальные тексты статей и советов (Q2 — позже).
- Реальные курируемые фото интерьеров для квиза и галерея «до/после» (content-later; квиз — на
  плейсхолдерах). Источник картинок/лицензии — решение владельца.
- Настоящий вход по логину/паролю (полноценный кабинет — позже по мастер-плану).
- Любые правки ядра сметы/прайсинга, AI-пайплайна, платежей.
- Глубокое SEO (только базовые `title`/`description` у новых страниц).
- Юридическое (ПДн/cookie) — TODO по CLAUDE.md, не блокируемся.
