## Цель
UI-доводка `/calc/plitka`: верхняя кнопка «найдём выгоднее» (текст/цвет/понятный переход), ровная
раскладка проёма, отделить «+ добавить размеры пола» от параметров стены, зелёные кнопки размеров.

## Источник задачи
Промпт владельца по `/calc/plitka` (4 пункта) + приглашение задавать вопросы.

## Ключевой контекст (по коду)
- Цвета шапки: «Посчитать материалы» = `.nav-cta` (терракота `--accent-strong`); «Сколько стоит
  ремонт» = `.nav-cta--alt` (зелёный `--accent`).
- Верхняя кнопка сейчас: текст «Найдём выгоднее — подберём материалы дешевле», зелёная, по клику
  `scrollIntoView` к `#find-cheaper`. Проблема владельца: после прокрутки не виден акцент — не ясно,
  что делать (блок `FindCheaper` в свёрнутом виде, кнопка «Найти выгоднее» неприметная).
- Проём (`SurfaceEditor`): Ш/В в одном ряду с кнопкой «×» → 3 колонки, поля уже стеновых (2 колонки) → «съезжание».
- `RoomPanel` (плитка без пола): «+ добавить размеры пола» идёт СРАЗУ после `MaterialParams` стены (без разрыва).

## Скоуп — что входит
1. **Верхняя кнопка**: текст → «Если есть ссылка на товар — найдём выгоднее»; цвет → терракота
   (как «Посчитать материалы», `--accent-strong` = дефолтный `.btn`). По клику: не просто скролл, а
   **открыть форму** блока `FindCheaper` (сразу видно поле ссылки — «куда нажимать») + плавный скролл
   к блоку (по центру) + короткая **подсветка-пульс** блока (~1.4с), чтобы взгляд попал на него.
   Реализация: проп `openSignal` у `FindCheaper` (ref + `useEffect`: `setOpen(true)` + scrollIntoView +
   класс `.pulse-highlight`); верхняя кнопка инкрементит сигнал.
2. **Раскладка проёма** (плитка/краска): проём — отдельный под-блок под размерами стены, как у стены:
   строка-заголовок «Проём» + «удалить», ниже Ширина/Высота в двух РАВНЫХ колонках (flex:1, minWidth 100),
   ровно под полями стены, без съезжания (кнопку удаления вынести в заголовок, а не в ряд полей).
3. **Разрыв перед «+ добавить размеры пола»**: `<div className="divider" />` перед кнопкой пола (и в
   ветке «пол ещё не добавлен»), чтобы отделить от «Параметры материала» стены.
4. **Зелёные кнопки размеров**: «+ добавить размеры стены» (`SurfaceEditor`) и «+ добавить размеры пола»
   (`RoomPanel`) — фон `--accent` + текст `--surface` (как «Сколько стоит ремонт»/«Заполнить»).

## Скоуп — что НЕ входит
- Логика расчёта, формулы, контракты — не трогаем (чистый UI).
- Прочие кнопки-chip («+ добавить проём», «+ площадь»/«- площадь», вкладки комнат) — цвет не меняем.
