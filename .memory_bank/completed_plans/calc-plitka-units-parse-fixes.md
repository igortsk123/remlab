## Цель
Починить «неправильный расчёт» плитки (корень — единицы: люди вводят размер в СМ, поле в мм),
корректно парсить размер и цену с сайтов (см, разделитель `*`, дробные; цена за м²/шт, не за упак),
и причесать UI: чистый результат без дублей, плейсхолдеры «0», зоны пола в строку, тексты.

## Источник задачи
Промпт владельца по /calc/plitka (8 пунктов + пример «Пол 20.06 м² → 2848 шт при плитке 60×120»,
диагноз владельца: «указывают в см, а у нас мм»). Уточнения (AskUserQuestion):
- Цена: **поле цены + выбор единицы** (за м² / за упаковку / за шт), парсер подставляет единицу.
- Плейсхолдер «0» светло-серым — **на всех числовых полях материала**.
- «Считает неправильно» — тот же корень (единицы/парсинг), отдельного примера с ручным вводом нет.

## Диагностика (подтверждена на реальных страницах)
- **Единицы плитки:** поле «Длина/Ширина плитки, мм», а размеры плитки в РФ указывают в СМ
  (60×120, 20×20, 7,4×15). Ввод «60×120» → 6×12 см → 2848 шт вместо ~31. → **поле плитки в СМ**.
- **Kerama** (`shop.kerama-marazzi.ru`): в теле `Размер: 7,4*15 см` (разделитель `*`, запятая, «см»);
  цена `2 246 руб./кв.м` (за м², есть и `186 руб./шт`). Regex не ловит `*`/дробные, размер берётся
  только из title, цена кладётся в pricePerPackRub. 
- **Керамогранит** (`keramogranit.ru`): размера в title нет, в теле `Размер (см) 60x120` → 600×1200 мм;
  цена `4790 руб./м2` (за м²).

## Скоуп — что входит
1. **Единицы плитки → СМ.** Поля «Длина/Ширина плитки, **см**» в MaterialParams: хранение остаётся в мм
   (контракт/формулы не трогаем), конверсия на границе UI (см↔мм). Ламинат/панели — остаются в мм.
2. **Чистый результат (PartNote, RoomPanel):** убрать дублирующую строку note при неизвестном размере;
   пустая площадь (0 м²) → одна му트-строка «Введите размеры {стен/пола}»; для пола parenthetical
   «(без проёмов…)» → «(без учёта исключённых зон; всего Y м²)».
3. **Кнопки add:** свести «+ добавить размеры стены» и «+ добавить размеры пола» в один общий стиль
   (на проде уже идентичны — фиксируем, чтобы не разъезжались).
4. **Парсинг цены + единица:** детект `руб./кв.м|м2|м²` → pricePerM2Rub, `руб./шт` → pricePerPieceRub,
   иначе pricePerPackRub. В форме — одно поле «Цена, ₽» + селектор единицы (за м²/упаковку/шт),
   парсер подставляет единицу. Новое поле контракта `pricePerPieceRub`; формула tile — ветка цены за шт.
5. **Парсинг размера:** читать «Размер …» из тела (не только title); поддержать разделители `x × х *`,
   дробные (7,4), единицу см/мм; эвристика без единицы: оба ≥100 → мм, иначе см. Конверсия в мм.
6. **AI-фолбэк:** добавить в FIELDS/KEY_FIELDS плитки `pricePerM2Rub`, `pricePerPieceRub` (описания
   единиц), чтобы ИИ клал цену в верную единицу.
7. **Плейсхолдеры «0» светло-серым** на всех числовых полях материала (MaterialParams) + CSS `::placeholder`.
8. **FloorEditor:** «Исключить зоны» → «Исключить зону»; зоны (доп. и исключаемые) — длина+ширина
   В СТРОКУ (сейчас `width:100%` в `.row` роняет их в столбик), крестик × справа с краю, без подчёркивания.

## Скоуп — что НЕ входит
- Смена единиц хранения в контракте/формулах (мм остаётся внутренним; конверсия только в UI).
- Ламинат/обои/краска единицы — не трогаем (панели в мм — норма).
- Проёмы (уже убраны прошлым ADR-0027).
- Идеальный парсинг всех магазинов — best-effort + ручная правка полей.
