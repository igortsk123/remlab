## Цель
Сделать так, чтобы в разборе генерации (`/trace N`) по КАЖДОМУ шагу цепочки были: дословный промпт,
дословный ответ модели и **кликабельные ссылки на картинки** (вход / промежуточные / выход),
которые владелец открывает в браузере. Первопричина — картинки вообще не сохраняются на проде.

## Источник задачи
Владелец разбирал генерацию №1 для диагностики «что не так». Ожидал по каждой модели: точный
отправленный промпт (слово в слово), точный ответ модели и ссылку на фото (исходник, промежуточные,
итог), открываемую в браузере. По факту: тексты есть, а картинок нет ни одной.

## Первопричина (подтверждена на проде, только чтение)
- App работает под `USER nextjs` (uid **1001**), а том трейсов `/app/data/traces` принадлежит
  `root:root`, `drwxr-xr-x` → uid 1001 **не может писать** (проверено: `touch` → Permission denied).
- `saveAsset` ([lib/trace/assets.ts](../../lib/trace/assets.ts)) на `fs.mkdir/writeFile` ловит EACCES
  и **молча** возвращает `null` (`catch { return null }`) → ни файла, ни строки в БД.
- Итог по всей БД: `generation_runs=1, generation_steps=3, generation_assets=0`; диск трейсов пуст.
- Тексты (промпты/ответы) пишутся через `recordStep` отдельно → не задеты, поэтому они есть.
- Вторая проблема: даже когда ссылки появятся, текущий asset-URL требует `?token=<TRACE_ADMIN_TOKEN>`
  — печатать такой токен в чат нельзя (утечка секрета). Нужны **подписанные (signed) ссылки** без
  раскрытия админ-токена.

## Скоуп — что входит
1. **Фикс прав на том трейсов** — чтобы `nextjs` (1001) мог писать; durable (переживает redeploy).
2. **Убрать «немой» глушитель** — `saveAsset` при ошибке пишет `console.warn` (без секретов), чтобы
   такой класс сбоев больше не прятался. Пайплайн по-прежнему не валим (best-effort сохраняем).
3. **Signed asset-ссылки** — HMAC(assetId+exp) на ключе `TRACE_ADMIN_TOKEN` (новый секрет не заводим):
   ссылка вида `/api/trace/asset/<id>?exp=<ts>&sig=<hmac>`, ограничена по времени, без раскрытия токена.
   Asset-роут принимает `sig` ИЛИ старый `token`/заголовок (обратная совместимость).
4. **Абсолютные ссылки в выводе `/api/trace/[seq]`** — отдавать полный `https://remont-lab.online/...`
   со `sig`, чтобы владелец открывал в браузере.
5. **Обновить скилл `/trace`** — формат ответа: по каждому шагу дословный промпт + дословный ответ
   модели + ссылки на картинки (input/intermediate/output). Цена/время — мелким, вторично.
6. **Верификация e2e** — после деплоя прогнать реальную генерацию, убедиться: строки ассетов есть,
   файлы на диске есть, signed-ссылка в браузере отдаёт картинку.

## Скоуп — что НЕ входит
- Бэкофилл картинок для генерации №1 и прочих до фикса — **невозможно** (байты не были захвачены).
  Ссылки появятся только для НОВЫХ прогонов. Явно сказать владельцу.
- Изменения продуктовой логики пайплайна (состав шагов, промпты, модели) — не трогаем.
- Юридика/ПДн-политика хранения фото — вне скоупа (TODO при необходимости).
- Ничего на VPN-ноде/общих конфигах хоста — только изолированный том `remlab-traces`.
