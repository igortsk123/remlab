## Ключевой контекст (по коду)
- Числовые поля: `strToNum`/`numToStr` (`lib/calc/num.ts`) round-trip через контролируемое value →
  промежуточный ввод «1,» стирается до «1» → дробь не набрать. Поля: `SurfaceEditor`, `FloorEditor`, `MaterialParams`.
- Проёмы: чекбокс «Учесть окна и двери» вверху `SurfaceEditor`; «+ добавить проём» — под-блок у стены.
- Глобальная шапка `SiteHeader` (sticky top:0, z-index 50), высота переменная (2 ряда).
- Итоги уже реактивны: `CalcBuilder` считает `computeRoomParts` на каждый ввод → липкая шапка получит их даром.

## Скоуп — Заход A
1. **Дробный ввод (. и ,)**: новый `components/calc/NumInput.tsx` — держит «сырую» строку, принимает
   `1.2`/`1,2`/`1,`/`1.` при наборе, наружу отдаёт число; ресинк при внешнем изменении (автозаполнение).
   Заменить сырые числовые `<input>` в `SurfaceEditor`, `FloorEditor`, `MaterialParams`. Проверить ВСЕ поля.
2. **Группировка проёмов**: чекбокс «Учесть окна и двери» → в тонированной рамке-группе с подсказкой
   («Проёмы задаёте у стен ниже»); под-блоки проёмов у стен — с акцентной левой границей (визуально связаны).
3. **Автоподстановка высоты**: новая стена наследует высоту ПЕРВОЙ стены; при вводе высоты первой стены
   заполнять высоту стен, где она ещё пустая (0). Каждую можно менять.
4. **Липкая шапка итогов**: компактная строка под `SiteHeader` (sticky), «Площадь: X м²» + «≈ Y ₽»
   (стоимость — только если задана цена). Живой пересчёт. Смещение — CSS-переменная `--site-header-h`,
   публикуемая `SiteHeader` (ResizeObserver на `documentElement`).
