## Цель
Переорганизовать /calc/plitka в две симметричные секции — «Плитка для стен» и «Плитка для пола».
Обе ЗАКРЫТЫ по умолчанию; пользователь открывает нужную (обе или одну) — раскрывается блок ссылки +
размеры + результат. Над блоком ссылки — неброский подзаголовок секции (чтобы не путать стены/пол,
но не отвлекать от «Вставьте ссылку»).

## Источник задачи
Промпт владельца + ответ на вопрос: «сначала обе закрыты, человек может открыть все, либо одну».
Плюс: подзаголовок секции над «Вставьте ссылку» (стены/пол); текст «(задайте размер плитки или
вставьте ссылку)» → «(вставьте ссылку или задайте размер плитки)».

## Дизайн (согласовано)
Симметрично текущему полу — но и для стен:
- Закрытая секция = зелёная кнопка `+ Плитка для стен` / `+ Плитка для пола`.
- Открытая секция = подзаголовок «Плитка для стен/пола» (неброский eyebrow) + «убрать» → блок ссылки
  (`LinkAutofill`, свой prominent CTA) + размеры + инлайн-результат.
- «убрать» сворачивает И очищает данные этой секции (стены: surfaces/material/productUrl; пол:
  floor/floorMaterial/floorProductUrl) — как сейчас у пола.
- Открыто по умолчанию, если в секции УЖЕ есть данные (сохранённый проект): стены — surfaces>0 ||
  productUrl || есть material; пол — есть room.floor. Свежая комната → обе закрыты.

## Скоуп — что входит
- Только ветка `kind === "plitka"` в `RoomPanel.tsx`: две секции с состоянием открыт/закрыт.
- Стены: локальный стейт `wallsOpen` (дефолт по наличию данных); кнопка-открытие; подзаголовок +
  «убрать» при открытии; внутри — wallLink, wallSizes (если surfaces>0), результат, «+ размеры стены».
- Пол: как сейчас (room.floor как маркер), но оформить симметрично — подзаголовок «Плитка для пола» +
  «убрать», кнопка-открытие «+ Плитка для пола».
- `CalcBuilder.tsx`: добавить `key={active.id}` у `RoomPanel` (сброс локального стейта при смене комнаты).
- Текст qtyUnknown: «(вставьте ссылку или задайте размер плитки)».

## Скоуп — что НЕ входит
- Другие виды (oboi/kraska/laminat) — оставить как есть (одна секция).
- Логика формул/парсинга/контрактов — не трогаем. Итоги считаются из данных (закрытая пустая секция
  = 0, не влияет).
