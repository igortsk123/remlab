## Цель
Довести UX и копирайт калькулятора обоев (`/calc/oboi`): человечные тексты, убрать ручной
ввод проёмов (заменить подсказкой-«окном»), правильные склонения «рулон», визуальные разрывы
и новый порядок хвостов (не забудьте → выгоднее → визуализация). Соседние калькуляторы
(плитка/краска/ламинат) остаются грамматически корректными (общие компоненты).

## Источник задачи
Промпт владельца по странице https://remont-lab.online/calc/oboi — 9 правок: заголовок,
кнопка «+ стена», убрать проёмы + подсказка-окно, текст ссылки на товар, разрыв перед ссылкой,
склонения «рулон» в Итоге, блок «Также не забудьте» (чек-лист), «Найдём выгоднее», «Как это будет выглядеть».

## Ключевой контекст (почему так)
- Роут `app/calc/[kind]/page.tsx` — ОБЩИЙ для oboi/plitka/kraska/laminat; компоненты
  `SurfaceEditor`/`LinkAutofill`/`FindCheaper`/`VizCta` тоже общие. → тексты параметризуем по `kind`
  через `CALC_META`, чтобы у соседей не сломалась грамматика.
- **Проёмы для обоев формула игнорирует** (`lib/calc/formulas.ts` → `wallpaper()` считает по периметру
  и макс. высоте). Для плитки/краски проёмы ВЫЧИТАЮТСЯ из площади (`netM2`). → убрать проёмы можно
  ТОЛЬКО для `kind === "oboi"`; у плитки/краски UI проёмов не трогаем.
- Формулы (`lib/calc/formulas.ts`) и их golden-тест НЕ трогаем — склонения только в отображении.
- Дизайн-токены/классы — `app/globals.css` (`.card .stack .row .eyebrow .chip .note .muted .btn`).

## Скоуп — что входит
1. **Заголовок:** `Сколько нужно: обои` → `Сколько нужно обоев` (родит. падеж, по всем видам через `titleGen`).
2. **Кнопка добавления стены:** `+ стена` → `добавить размеры стены` (в `SurfaceEditor`, для всех видов со стенами).
3. **Убрать проёмы (только oboi):** снять строки проёмов + кнопку `+ проём`; вместо неё справа —
   маленькая приглушённая иконка-окно с всплывающей подсказкой (hover + focus/тап):
   «При расчёте окна и двери не учитываются. Это сделано намеренно, так как расход обоев
   определяется количеством целых полос. Проёмы в большинстве случаев не уменьшают необходимое
   количество рулонов.» У плитки/краски проёмы остаются как есть.
4. **Текст ссылки на товар:** `Ссылка на товар — заполним параметры сами` →
   `Добавьте ссылку на выбранные вами обои. Мы заполним параметры сами.` (по видам через `accYour`).
5. **Визуальный разрыв** в `RoomPanel` между блоком размеров стен и блоком ссылки на товар (divider).
6. **Склонения «рулон» в Итоге:** `1 рулон / 2 рулона / 5 рулонов` — новый хелпер `plural`/`pluralUnit`;
   применить в `ResultView` (Итог) и `RoomPanel` («Нужно: …») для консистентности. Безопасно для всех
   единиц (м²/л/шт не меняются, упаковка — тоже склоняется).
7. **«Также не забудьте:»** (был «Заодно не забудьте») — вывести companions чек-листом с галочками
   (✓) вместо чипов; переставить сразу ПОСЛЕ Итога.
8. **«Найдём выгоднее»** (`FindCheaper`, был «Найти дешевле») — заголовок + текст:
   `Мы подберём те же обои и все необходимые материалы для поклейки по более выгодной цене — со
   скидками, в магазинах вашего города или онлайн.` (по видам через `accPick` + `work`).
9. **«Как это будет выглядеть»** (`VizCta`) — текст:
   `Загрузите фото комнаты и посмотрите, как будет выглядеть ваша комната после обновления отделки.`
10. **Порядок хвостов** в `CalcBuilder`: Итог → Также не забудьте → Найдём выгоднее → Как это будет выглядеть.

## Скоуп — что НЕ входит
- Формулы расчёта (`lib/calc/formulas.ts`, `lib/estimate/calc.ts`) и их golden-тесты — не трогаем.
- Логика проёмов для плитки/краски — без изменений (только скрываем UI для обоев).
- Оплата визуализации, серверный поиск «дешевле», лид-магнит — поведение не меняем (только копирайт/порядок).
- Юридическая логика согласий — не трогаем.
