# User Flow — детали (Tier 2 к [[user-flow]])

> Tier 1: `../core/user-flow.md`. Полный CJM (все состояния, события, acceptance criteria) —
> `../../docs/cjm-ux-v0.2.md`. Сверено с кодом 2026-07-09.

## Укороченный Stage 1 flow (7 экранов, целевой дизайн)
1. **Landing** — за 5с объяснить пользу; пример «фото→preview→товары(3 из 18)→бюджет→CTA».
   CTA: «Обновить комнату», «Рассчитать стоимость».
2. **Комната+цель+уровень** (один экран): гостиная/спальня; обновить/понять стоимость;
   уровень 1 освежить / 2 недорого обновить / 3 лёгкий косметический.
3. **Фото+бриф** (один экран): 1–3 фото, город, бюджет-диапазон, ограничения.
   Не спрашивать размеры/розетки до preview.
4. **Style cards**: фото + название стиля + акцент + объяснение; достаточно 3–5. → `style_profile`.
5. **AI-preview + подбор до N**: «AI понял комнату», 1 визуальный вариант (+дисклеймер «концепция,
   товары подобраны похожие»), 3–5 идей, **до N (3–5) РЕАЛЬНЫХ товаров из фидов с ценами и
   реф-ссылками, открыто** (+2–3 альтернативы), товарный бюджет (сумма цен).
6. **Paywall / room pack**: показать структуру платного, закрыть детали. После оплаты: ВСЕ элементы
   комнаты (8–20) + 3 альтернативы, цены/реф-ссылки, **Cost Engine** (стоимость работ), чек-лист,
   план, PDF, сохранение.
7. **Workspace «Мои комнаты»**: объект→комнаты→версии; обновить цены, заменить недоступные, PDF.

## Free vs Paid — 3 варианта (owner 2026-07-02, детали)
1. **«Освежить без ремонта» — БЕСПЛАТНО:** AI-визуализация + подбор **до 3 реальных предметов мебели**
   по визуализации (открыто: фото/цена/реф-ссылка + 2–3 альтернативы, можно заменить) + товарный
   бюджет. Без материалов/сметы/плана.
2. **«Недорого обновить» — ПЛАТНО ~1 490 ₽:** мебель **в любом количестве** + подбор **материалов**.
   Без расчёта стоимости ремонта, без чертежей/дизайнера. Визуализация + реф-ссылки.
3. **«Ремонт под ключ» — ПЛАТНО 9 900 ₽:** мебель + материалы + гайды + инфо для строителей
   (**чертежи**) + **смета (Cost Engine)** + **услуга живого дизайнера**. Замена дизайнера,
   но в 5–10× дешевле.
- **Реф-ссылки во всех вариантах** (affiliate). **Paywall-триггер:** выбор варианта 2 или 3
  (НЕ доступ к 3 бесплатным товарам — они всегда открыты и кликабельны).

### Код-долг по границе (сверено с кодом 2026-07-09)
- Структура «3 открыто, остальное закрыто» есть: `FREE_VISIBLE = 3` в `modules/ideas/index.ts`
  (верна для варианта 1), но каталог — **seed** (`buildCatalog()` из SEED-массива).
- Нужно: реальные товары из фидов Гдеслон + реф-ссылки; варианты 2/3 (материалы, любое кол-во,
  чертежи, Cost Engine, дизайнер); согласовать с `interventionLevel`.
- Оплата — **заглушка**: `unlockPack` в `app/actions.ts`, на paywall-экране явный «Демо-режим»;
  настоящая оплата (YooKassa) — отдельным шагом.

## Экраны в коде (сверено 2026-07-09)
- `/` (`app/page.tsx`) — лендинг: CTA «Обновить комнату» → `/start`, «Рассчитать стоимость» →
  `/soon?f=cost`; ссылка «Мои комнаты» → `/rooms`.
- `/start` → создание проекта; `/p/[id]/brief` → фото+бриф; `/p/[id]/select` → интерактивный выбор;
  `/p/[id]/preview` → превью+идеи+товары; `/p/[id]/paywall` → демо-paywall.
- `/rooms` — workspace «Мои комнаты» (список проектов сессии, статусы started…paid).
- `/soon` — fake-door «Расчёт стоимости ремонта» (сбор email, событие `cost_fake_door_viewed`).
- `/p/[id]/style` — редирект на `/p/[id]/select` (экран стиля перенесён туда).

### Интерактивный выбор (`/select`, план `interactive-object-selection`)
- На монтировании — `runAnalyze` (`modules/generation-job`): разбор фото на объекты, отдельный
  ПРЕД-шаг вне `runWithTrace` (без номера генерации), идемпотентен (повторный заход не жжёт вызов
  модели). Дефолт `objectChoices` = предложение модели.
- Пользователь по каждому объекту: **оставить / поменять / убрать** (`action` = keep/change/remove)
  + выбирает **стиль** + **одно поле-пожелание** (`wish`, свободный текст).
- «Сгенерировать» → `runGenerate`: restyle с выбором пользователя в промпте + ideas, обёрнут в
  `runWithTrace` — это и есть «генерация» с номером. Проверяемо в `/trace`: промпт restyle содержит
  «Поменять/Оставить/Убрать: …» + пожелание.
- Кнопки «что оставить» в брифе убраны (заменены пообъектным выбором); новое фото сбрасывает
  analysis+objectChoices (`app/actions.ts`).

## Второй вход: «найти похожую мебель по фото» (план)
Короткий flow без генерации: до N элементов с фото (своё/найденное) → прямой поиск похожих в базе
фидов (pgvector). Самостоятельная точка входа, технически проще основного flow.

## Сегмент F — дизайнер интерьера (B2B, Stage 1.5, план)
Опт: подбор аналогов пачками. Нужны batch-режим (несколько комнат), выгрузка таблица/PDF, без
брендинга в клиентских материалах (white-label позже). Отдельный CJM. Цена ~990 ₽/комната от 10.

## Cost-сценарий (Stage 1B, план)
Известные данные → недостающее (площадь, потолок, состояние стен/пола, уровень материалов) → AI
предлагает scope → пользователь подтверждает → **Cost Engine** считает диапазон + структура +
confidence (low/mid/high). Сумму считает движок, НЕ LLM.

## Аналитика
- **В коде** (`lib/analytics.ts`): серверный PostHog (free-тариф; `POSTHOG_KEY` не задан → no-op;
  хост `POSTHOG_HOST`, дефолт eu). События: `project_started`, `brief_completed`, `style_selected`,
  `preview_ready`, `paywall_viewed`, `pack_unlocked`, `cost_fake_door_viewed`, `problem_reported`,
  `app_error` (+ `captureError`). Ошибки тоже через PostHog; Sentry — позже, если захотим.
- **План (CJM, НЕ реализовано):** события по всем экранам (landing_*, room/goal/intervention,
  photo/brief, style_card_*, ai_preview_*, paywall_*, payment_*, workspace_*, cost_*) +
  **affiliate** (`affiliate_link_clicked`, `affiliate_postback_received`,
  `affiliate_purchase_attributed`) + **лимиты генерации** (anti-abuse анонимов).
  Полный список — `../../docs/cjm-ux-v0.2.md`.
