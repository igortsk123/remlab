# User Flow — Tier 1 сводка

## Укороченный Stage 1 flow (7 экранов)
1. **Landing** — за 5с объяснить пользу; пример «фото→preview→товары(3 из 18)→бюджет→CTA». CTA: «Обновить комнату», «Рассчитать стоимость».
2. **Комната+цель+уровень** (один экран): гостиная/спальня; обновить/понять стоимость; уровень 1 освежить / 2 недорого обновить / 3 лёгкий косметический.
3. **Фото+бриф** (один экран): 1–3 фото, город, бюджет-диапазон, «что оставить», ограничения. Не спрашивать размеры/розетки до preview.
4. **Style cards**: фото + название стиля + акцент + объяснение; достаточно 3–5. → `style_profile`.
5. **AI-preview + ограниченный результат**: «AI понял комнату», 1 визуальный вариант (+дисклеймер «концепция, проверьте размеры»), 3–5 идей, 2–3 preview товара, бюджет-диапазон (+confidence если Cost Engine).
6. **Paywall / room pack**: показать структуру, закрыть детали. После оплаты: полный список товаров, 3 альтернативы, цены/ссылки, бюджет, чек-лист, план, PDF, сохранение.
7. **Workspace «Мои комнаты»**: объект→комнаты→версии; обновить цены, заменить недоступные, PDF.

## Free vs Paid
- **Free:** доказать что продукт понял комнату + польза, но НЕ выдавать полный список/цены/альтернативы/PDF.
- **Paywall-триггер:** после AI-preview и ограниченного preview товаров (не раньше — не увидел ценность; не позже — платить слабее).

## Cost-сценарий (Stage 1B)
Известные данные → недостающее (площадь, потолок, состояние стен/пола, уровень материалов) → AI предлагает scope → пользователь подтверждает → **Cost Engine** считает диапазон + структура + confidence (low/mid/high). Сумму считает движок, НЕ LLM.

## Аналитика
PostHog события по всем экранам (landing_*, room/goal/intervention, photo/brief, style_card_*, ai_preview_*, paywall_*, payment_*, workspace_*, cost_*). Полный список — Tier 2.
