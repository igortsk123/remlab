# Стандарты кода (TypeScript + Zod по умолчанию)

> Тонкая версия для авто-загрузки. Полный чеклист — `guides/code-standards.md`.
> Дефолт — **TypeScript + Zod** (соло-вайбкодинг: меньше настройки на старте). Не-TS проект:
> `/memory-init` подстроит `paths:` и примеры под обнаруженный стек.

## TypeScript
- `strict: true` в tsconfig. Без `any` / неявных типов. Публичные интерфейсы типизированы явно.
- Named exports (не default) — лучше для поиска/рефакторинга.
- Тип выводим из источника правды (Zod-схема → `z.infer`), не дублируем руками.
- Без сверхсложных generic без необходимости — простые понятные типы.

## Zod на границах доверия (по умолчанию)
- Любой ВНЕШНИЙ вход — Zod-DTO: API/route handlers, server actions, формы, парсинг файлов, env.
- `schema.safeParse()` на входе → дальше работаем с провалидированным DTO; тип — `z.infer`.
- Простые ПЛОСКИЕ схемы, минимум магии. Бизнес-правила НЕ в схеме (отдельный слой/доменка) —
  Zod валидирует ФОРМУ/типы, не «обязательно-если-применимо» и пр.
- ⚠️ Гоча: `z.preprocess` срабатывает и на ОТСУТСТВУЮЩЕМ ключе (`undefined == null`) — для
  частичных патчей это молча СТИРАЕТ поля. Хелперы делать без `preprocess`; тестировать частичный
  патч (один ключ на входе → в выходе только он).

## Слои и именование
- Сетевые вызовы / доступ к данным — в выделенном слое (`services/`/`lib/`), не в UI-компонентах.
- Компоненты/классы — PascalCase; функции/переменные/хуки — camelCase; константы — SCREAMING_SNAKE.

## Запреты / лимиты
- Дублирование >10 строк → вынести. Логика в разметке/шаблоне. Оставленный `console.log`.
- Компонент >150 строк / модуль/сервис >100 строк → разбить по ответственности.

## Состояния экрана (UI)
Покрыть `loading` / `error+retry` / `empty` / `success` — не только happy-path.
