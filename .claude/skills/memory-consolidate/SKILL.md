# memory-consolidate — свод сессии в Memory Bank

Цель: чтобы в `.memory_bank/` (постоянная иерархичная память проекта, коммитится в git) оказалось
ВСЁ проектное, что узнали за сессию, — **пока локальная/сессионная память Клода не очистилась**.
Запускать в конце сессии или когда попросят. Это и есть механизм, который не даёт истории проекта
осесть только в служебной памяти harness и потеряться.

Правила-основа: `.claude/rules/memory-discipline.md`, навигация `.memory_bank/INDEX.md`.

## Источники
1. **Текущая сессия** (диалог/транскрипт) — основной источник.
2. **Локальная авто-память harness** (если включена): `~/.claude/projects/<sanitized-cwd>/memory/` —
   перечитать свежие записи и перенести проектное в `.memory_bank/`. Если папки нет — пропустить.

## Что вытаскивать (durable, ПРОЕКТНОЕ)
- Архитектура / новые слои / паттерны.
- Внешние интеграции и доступы: эндпоинт, ГДЕ лежит ключ/прокси, формат, клиент в коде (без значений секретов).
- Принятые решения (+ почему / что отвергли).
- Важные цифры (метрики, базлайны).
- Ход работы / вехи (что сделано, что отложено, статусы планов).
НЕ брать: значения секретов, тривиально выводимое из кода, спекулятивное, НЕ относящееся к проекту.

## Процедура
1. **Собрать** кандидаты-факты из источников.
2. **Сверить с памятью** (anti-dup): для каждого — `grep` по `.memory_bank/`. Уже есть → пропустить.
3. **Записать** недостающее по таблице memory-discipline:
   - решение → `decisions.md`; статус/ход → `project-state.md`; интеграция/доступ → `core/access-and-integrations.md`;
   - сводка темы → `core/<topic>.md`; глубина → `<area>/<topic>.md`; веха плана → `plans/`/`completed_plans/`.
   - **Только ДОПОЛНЯТЬ**; существующее не переписывать/не удалять. Датировать `[YYYY-MM-DD]`.
4. **No-orphan:** каждый НОВЫЙ док в той же правке внести в `INDEX.md` (decision tree / реестр).
5. **Безопасность:** не трогать `plans/*` (draft/in_progress), `_intake/*`, `completed_plans/*`, canonical-доки
   без необходимости; секреты не выносить.
6. **Сверка:** прогнать `/memory-check` (структурный гейт: orphan/stale/битые ссылки, INDEX).
7. **Лог:** одна строка в `.memory_bank/changelog/memory-log.md` (дата, что сведено).
8. **Отчёт:** что дописал / что уже было / что вынес в INDEX / результат memory-check.

## Принципы
- Свод, а не накопление: один факт — один источник, остальное ссылается (`[[name]]`).
- Сомнение «проектное ли» → если про проект, писать; иначе пропустить.
- Это ручной/по-флагу свод (безопасно). Авто-хук на каждый SessionEnd НЕ ставим (рекурсия/стоимость):
  вместо этого Клод сам запускает этот скилл в конце сессии (см. `memory-discipline.md` → «Цикл сессии»).
