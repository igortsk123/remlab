# /trace — разбор генерации по номеру

Цель: по номеру генерации быстро показать владельцу, что происходило в пайплайне — в двух видах
(сначала краткий, потом подробный) — и прислать исходное и итоговое фото (+ промежуточные, если были).
Владелец не кодер: пиши по-человечески, без жаргона; технические id оставляй в подробном блоке.

## Вход
Номер генерации `N` (целое) из фразы пользователя. Если номера нет — спроси один короткий вопрос.
Иногда просят только краткий или только подробный вид — уважай это; по умолчанию давай оба.

## Где данные
- **Прод (по умолчанию):** генерации пользователь гоняет на https://remont-lab.online. Разбор — через
  admin-роут `GET /api/trace/<N>` (JSON: run+steps+assets). В JSON каждая картинка уже приходит с
  `url` — **абсолютной подписанной ссылкой** (`?exp=..&sig=..`), которую владелец открывает в браузере;
  admin-токен в неё НЕ входит. Сам запрос к `/api/trace/<N>` требует `TRACE_ADMIN_TOKEN` (в `/opt/remlab/.env`
  на сервере): выполняй его ПРЯМО НА СЕРВЕРЕ по ssh, подставляя токен из env (`?token=${TRACE_ADMIN_TOKEN}`),
  **не печатая значение токена в чат** — см. `.memory_bank/deployment.md`. В репозиторий/память токен НЕ писать.
- **Локально:** если работаешь против локальной БД — `pnpm trace <N>` (скрипт `tools/trace-show.mjs`,
  печатает всё + пути к файлам в `TRACE_DIR`, по умолчанию `./.data/traces`).

## Как отвечать
1. Получи данные (curl admin-роута на проде ИЛИ `pnpm trace N` локально).
2. **КРАТКО** (3–5 строк): статус, путь `шаг[модель] → шаг[модель] → …`, и — если был сбой — на каком шаге.
   Плюс словами: что на входе, что на выходе. Время/стоимость — вторично, одной строкой мелким (оценка).
3. **ПОДРОБНО — суть диагностики. По каждому шагу цепочки:**
   - модель (провайдер/модель), id+версия промпта;
   - **промпт ДОСЛОВНО** — точный текст `promptText`, слово в слово, а НЕ пересказ (в блоке кода);
   - **ответ модели ДОСЛОВНО** — точный `outputText`, если шаг вернул текст (в блоке кода);
   - **ссылки на картинки этого шага** — из `assets` (сгруппируй по `stepId`): вход (`role=input`),
     промежуточные (`role=intermediate`), выход (`role=output`). Дай кликабельный `url` как есть —
     владелец открывает в браузере. Так по КАЖДОЙ модели видно вход→выход и где проблема;
   - настройки/время/стоимость/ошибка — если есть.
4. Если у шага нет ожидаемой картинки (`assets` пусто) — скажи прямо: «не сохранена» и, если знаешь,
   почему (напр. прогон до фикса прав тома — картинки не захватывались). Не выдумывай ссылку.
5. Если генерация не найдена — скажи прямо и предложи проверить номер.

## Важно
- Трейс содержит фото пользователя (ПДн) — не публиковать наружу, показывать только владельцу в этом чате.
- **Токен в чат не печатать.** Ссылки на картинки — только подписанные (`?sig=`), не `?token=`.
- Ничего не удаляй и не меняй — это диагностика, только чтение.
- Числа стоимости — оценка (не биллинг), так и говори.
